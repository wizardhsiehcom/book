# MIT 14.12《Economic Applications of Game Theory 賽局理論的經濟應用》成書計畫

本計畫用來把 `data/mit14-12/clean/` 內的 MIT 14.12 課程逐字稿，整理成一本繁體中文的知識書。逐字稿為清理過的純文字 `.txt`（含標點、單行無換行），共 25 講，另有 `data/mit14-12/raw/` 內對應的 `.vtt` 原始字幕作為次要參照（僅在 `.txt` 疑似清理出錯時回查，不作為主要閱讀來源）。課程內容涵蓋個體決策、賽局表示、優勢與可理性化、Nash 均衡、動態賽局與子賽局完美、重複賽局與 Folk 定理、貝氏賽局、拍賣理論、訊號傳遞與 cheap talk、以及共同知識。核心原則是：每一講都必須完整閱讀逐字稿，從第一個字到最後一個字，不以片段、摘要、搜尋命中或抽樣段落代替完整閱讀。

## 目標

- 把課程內容整理成一本可在本 repo 內發展為 MkDocs book 的繁體中文書稿。
- 保留課程主線：從「什麼是賽局理論」與個體決策基礎出發，建立靜態賽局的解概念（優勢、可理性化、Nash 均衡），套用到寡占競爭與零和賽局；再進入動態賽局（逆向歸納、子賽局完美、一次偏離原理、議價）、重複賽局（有限、無限、Folk 定理、隱性卡特爾）；然後是不完全訊息（貝氏賽局、貝氏 Nash 均衡與應用、拍賣、收益等價、廣告拍賣）、動態不完全訊息（完美貝氏均衡、訊號傳遞、不完全訊息議價、cheap talk），最後收在共同知識。
- 將逐字稿中的口語講解（含課堂 MobLab 遊戲、板書推導、學生問答）轉成可閱讀、可複習、可交叉引用的書面章節。
- 最後補入外部對各主題的理解、經典文獻（如 Nash、Myerson、Aumann 的原始貢獻）或延伸資源，但只補充有助於理解課程主軸的資訊。

## 非目標

- 不直接將逐字稿全文貼成書稿。
- 不在未完整閱讀逐字稿前，先依課名或賽局理論常識寫出最終章節。賽局理論是成熟領域，worker 特別容易憑既有知識腦補——必須以逐字稿實際講過的內容為準。
- 不用外部文章或教科書取代課程逐字稿本身。
- 不為了速度跳過課堂遊戲、口誤、學生問答或重複段落；這些內容仍要讀完，再判斷是否進入書稿。
- 不在資訊不足時硬補內容。若本地沒有、使用者未提供、工具無法可靠取得，必須標記 `待補` 並請使用者提供，不可編造講者姓名、開課年份、賽局矩陣數值、定理敘述細節、習題、檔案路徑或資源連結。

## 資訊不足處理規則

本專案允許佔位，不允許猜測。遇到下列狀況時，主控 agent 或 worker 必須停止補寫，改以 `待補`、`待查` 或「需使用者提供」標示：

- 講者姓名與開課學期：逐字稿開頭未自報，目前標 `待查`，第 6 階段由 OCW 或影片來源確認。
- 課程無 slides、板書照片、syllabus、reading 清單、習題：`data/mit14-12/` 只有逐字稿與字幕。
- 逐字稿提到板書上的賽局矩陣、賽局樹或圖形，但口語無法完整還原其數值與結構時：能從上下文可靠重建者重建並標明「依口語重建」；不能者標 `待補`。
- 講者引用的論文、人名、年份口語含糊，無法可靠對應時。

處理方式：

1. 在 plan 或 tracker 中保留佔位。
2. 明確寫出缺少什麼材料。
3. 若需要使用者協助，直接提出需要提供的清單。
4. 不因為章節或表格想保持完整而自行補出不存在的資料。

## 材料特性注意事項

- `.txt` 為主要閱讀來源；`.vtt` 僅在懷疑清理版漏字、斷句錯誤時回查，不得以 `.vtt` 取代 `.txt` 的完整閱讀。
- 檔名含全形冒號「：」，路徑必須完全照抄。
- 逐字稿為單行無換行，追蹤大小以 bytes 計。
- 課堂使用 MobLab 平台玩互動遊戲；遊戲規則與結果是講者建立直覺的核心素材，必須完整記錄，不可當旁支跳過。
- 賽局理論充滿矩陣、賽局樹與數學符號；口語轉寫容易失真（例如 payoff 數值、策略名稱），worker 要邊讀邊重建形式化表示，無法確定的數值標 `待查`。

## 工作階段

### 多 agent 執行制度

本書由一個主控 agent 管控進度，再派多個章節 worker agent 逐講完成內容。主控 agent 負責整體一致性，不把品質責任完全交給 worker。

#### 角色分工

| 角色 | 責任 | 可寫入範圍 |
|---|---|---|
| 主控 agent | 維護計畫、派工、審稿、更新追蹤表、統一術語與符號、處理跨章整合 | `plan/mit14-12/`、`docs/mit14-12-game-theory/`、`configs/mit14-12-game-theory.yml` |
| 章節 worker agent | 完整閱讀指定逐字稿，產出該講閱讀筆記與章節初稿 | 僅限被指派的 `notes/lecture-XX-*.md` 與 `XX-*.md` |
| 外部補充 agent | 在全部逐字稿初稿完成後搜尋外部理解與補充資料 | `references.md` 與指定章節的「外部補充」段落 |
| 審稿 agent | 檢查單章是否忠於逐字稿、是否跳讀、形式化重建是否可靠 | 原則上只回報問題，不直接改稿；必要時由主控 agent 整合 |

#### 派工原則

- 不一次派出所有章節 worker；每批建議 3 到 4 講，降低術語漂移與整合成本。
- 每個 worker 只負責 1 講；相鄰講次主題高度連續（如 L12–L13 重複賽局）時，最多可負責 2 講。
- worker 必須被告知：repo 內可能有其他 agent 或使用者的改動，不可 revert 或覆蓋非自己負責的檔案。
- worker 的任務必須具體包含：
  - 指定逐字稿完整路徑（照抄全形字元）。
  - 指定閱讀範圍為全文（單行檔案，從第一個字到最後一個字）。
  - 指定輸出閱讀筆記檔與章節檔。
  - 禁止網路搜尋，除非主控 agent 明確進入第 6 階段。
  - 禁止只根據課名、賽局理論常識或片段搜尋結果寫章節。
  - 若材料不足，必須回報缺口，不可自行腦補。

#### Worker 交付格式

每個章節 worker 完成後必須回報：

1. 已完整閱讀的逐字稿檔名與位元組數（單行檔，以 bytes 取代行數）。
2. 新增或修改的檔案清單。
3. 本講 5 到 10 個核心概念。
4. 與前後章需要主控 agent 注意的連結。
5. 不確定或需要主控 agent 複查的點（含「依口語重建」的矩陣／賽局樹清單）。
6. 是否使用外部資料（初稿階段應為「否」）。

#### 主控 agent 驗收條件

主控 agent 收到 worker 結果後，必須檢查：

- 閱讀筆記是否明確標示完整閱讀範圍。
- 章節是否有把逐字稿內容抽象成書面解釋，而不是逐字翻譯或鬆散摘要。
- 是否有跳過課堂遊戲、學生問答、講者澄清等看似旁支但可能重要的內容。
- 賽局矩陣與樹的重建是否標明依據；是否有把課本標準版本混進來冒充講課內容。
- 術語與符號是否和既有章節一致。
- 是否有新增未經允許的外部資料。
- 是否只改動被指派的檔案。
- 是否把不確定或缺失材料標成 `待補`／`待查`，而不是寫成已確認事實。

驗收後主控 agent 才能更新 `plan/mit14-12/mit14-12-game-theory-transcript-tracker.md` 的狀態。

### 第 0 階段：建立書籍骨架

1. 建立全書工作目錄 `docs/mit14-12-game-theory/`。
2. 建立章節命名規則、頁面模板、術語表、參考資料頁。
3. 先用課程清單建立暫定章節，不寫實質內容，只標註每章來源逐字稿。
4. 建立閱讀紀錄表，追蹤每份逐字稿是否已「完整讀完」、「抽象完成」、「章節補寫完成」、「外部補充完成」。

### 第 1 階段：課程級抽象

在逐講細讀前，先建立高層架構，但只能以課名、檔案清單與課程主題做粗略規劃：

1. 定義全書讀者：對經濟學、策略互動或決策科學有興趣，想系統學習賽局理論並看到經濟應用的讀者；假設具備基本微積分與機率常識，不預設經濟學背景。
2. 將 25 講分成幾個篇章：
   - 導論與個體決策（L1）
   - 靜態賽局與解概念：表示法、優勢、可理性化、Nash 均衡（L2–L5）
   - 靜態應用：不完全競爭、零和賽局（L6–L7）
   - 動態賽局：逆向歸納、談判、子賽局完美、一次偏離原理與議價（L8–L11）
   - 重複賽局：有限重複、無限重複、Folk 定理、隱性卡特爾（L12–L15）
   - 不完全訊息靜態：貝氏賽局、貝氏 Nash 均衡應用（L16–L17）
   - 拍賣：拍賣形式、收益等價、廣告拍賣（L18–L20）
   - 不完全訊息動態：完美貝氏均衡、訊號傳遞、不完全訊息議價、cheap talk（L21–L24）
   - 共同知識（L25）
3. 為每篇寫一段暫定導言，之後會根據完整閱讀結果修正。
4. 注意跨篇連結：解概念是一條逐步精煉的鏈（優勢 → 可理性化 → Nash → SPNE → BNE → PBE），書稿要讓讀者看見這條鏈，而不是 25 個孤立主題。

### 第 2 階段：逐字稿完整閱讀

每一講按順序處理，流程固定：

1. 開啟 `data/mit14-12/clean/` 內對應的逐字稿 `.txt`。
2. 從頭到尾完整閱讀，不跳段（單行長檔案，分段讀取但必須讀完全文）。
3. 邊讀邊記錄：
   - 本講主問題
   - 關鍵概念與解概念定義
   - 定理敘述與證明思路（如 Folk 定理、收益等價定理、一次偏離原理）
   - 賽局實例：矩陣、賽局樹、payoff 設定（依口語重建並標明）
   - 課堂 MobLab 遊戲的規則、結果與講者詮釋
   - 經濟應用場景（寡占、卡特爾、拍賣、議價、廣告）
   - 講者使用的直覺、例子、類比
   - 問答或容易被忽略的細節
   - 與其他講次的關聯
4. 疑似清理錯誤時回查 `data/mit14-12/raw/` 對應 `.vtt`。
5. 讀完後才產出該講的「抽象筆記」。
6. 抽象筆記完成後，再改寫成書稿章節。

### 第 3 階段：章節寫作

每章建議結構：

1. 章節導讀：這一章要回答什麼策略互動問題、引入什麼解概念或應用。
2. 核心概念：用書面語整理逐字稿中的主要內容。
3. 形式化與定理：定義、定理、證明思路，必要時補推導步驟。
4. 賽局實例與應用：課程實際講過的矩陣、樹、經濟場景，以表格或 Mermaid 重繪。
5. 常見誤解：逐字稿中講者特別澄清、問答中出現、或容易混淆的點。
6. 與前後章的連接：說明本章在解概念鏈上的位置。
7. 小結：用 5 到 10 個條列收斂本章。

### 第 4 階段：全書整合

逐章完成後，進行跨章整理：

1. 統一術語翻譯，例如 dominance、rationalizability、Nash equilibrium、subgame-perfect、one-shot deviation principle、folk theorem、Bayesian game、perfect Bayesian equilibrium、signaling、cheap talk、common knowledge、revenue equivalence 等；統一數學符號（策略、payoff、資訊集合的記法）。
2. 檢查重複概念是否需要前置說明或交叉引用（例如「均衡精煉」貫穿全書；議價在 L9、L11、L23 三度出現）。
3. 補上全書導論與結語。
4. 建立圖表清單：解概念關係圖、代表性賽局矩陣、賽局樹、拍賣形式比較表。
5. 檢查章節順序是否忠於課程，但必要時可在書稿中加入跨章導讀。

### 第 5 階段：參考資料整合

1. 若使用者補充 syllabus、reading 清單或指定教科書，逐章標示對應章節作為延伸閱讀，不直接摘抄。
2. `data/mit14-12/` 目前只有逐字稿與字幕，無 slides 或習題；若使用者補充官方材料，再依既有整合模式處理。
3. 不因為材料缺失而中止書稿進度；相關欄位標 `待補`。

### 第 6 階段：網路補充

所有逐字稿完成閱讀與初稿後，才進行外部搜尋。搜尋目標：

1. 官方 MIT OCW 14.12 課程頁（確認講者、學期、syllabus、reading 清單）。
2. 課程相關經典文獻（逐字稿中實際提到者優先，如 Myerson、Aumann 的定義出處）。
3. 針對某些主題的高品質解釋（如 Folk 定理、收益等價定理）。
4. 與課程主題直接相關的經典論文或官方資源。

補充原則：

- 外部內容只能補強理解，不取代逐字稿。
- 每則外部補充都要標註來源 URL、作者或網站、存取日期。
- 若外部說法與逐字稿不同，優先保留差異並說明脈絡，不直接混寫。
- 不收錄只有泛泛摘要、SEO 內容、無法驗證作者或與課程無關的資料。

### 第 7 階段：出版化整理

1. 將內容轉入 `docs/mit14-12-game-theory/`。
2. 建立 `configs/mit14-12-game-theory.yml`。
3. 補上首頁、目錄、術語表、參考資料、索引，並在 `index.html` 註冊書卡。
4. 執行 `./sync-assets.sh` 與 MkDocs build。
5. 檢查中文排版、數學式、Mermaid 圖、連結、章節導覽。

## 品質檢查清單

- 每一章都能追溯到一份或多份完整讀完的逐字稿。
- 閱讀紀錄表不可只標「完成」，必須包含完成日期與筆記檔連結。
- 每一章的重點不是單純翻譯，而是把課程內容改造成讀者能建立心智模型的書面解釋。
- 定義、定理、賽局實例不可只用口語描述，必要時要補形式化表示或圖表。
- 依口語重建的矩陣與賽局樹已標明重建依據；無法重建者標 `待補`。
- 解概念鏈（優勢 → 可理性化 → Nash → SPNE → BNE → PBE）在跨章導讀中連貫。
- 外部資料必須有引用紀錄。
- 全書術語與符號要一致。
