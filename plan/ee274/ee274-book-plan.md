# EE274《Data Compression》成書計畫

本計畫用來把 `data/ee274/transcripts/` 內的 Stanford EE274 (2023) 課程逐字稿，整理成一本繁體中文的技術書。核心原則是：每一講都必須閱讀完整逐字稿，從第一行到最後一行，不以片段、摘要、搜尋命中或抽樣段落代替完整閱讀。

## 目標

- 把課程內容整理成一本可在本 repo 內發展為 MkDocs book 的繁體中文書稿。
- 將逐字稿中的口語講解轉成可閱讀、可複習、可交叉引用的書面章節。
- 最後補入網路上他人對 EE274 的理解、筆記、討論或延伸資源，但只補充有助於理解課程主軸的資訊。

## 非目標

- 不直接將逐字稿全文貼成書稿。
- 不在未完整閱讀逐字稿前，先依課名或既有常識寫出最終章節。
- 不用外部文章取代課程逐字稿本身。
- 不為了速度跳過重複、口誤、問答或示範段落。
- 不把作業 handout、lecture code、slides 直接塞進章節主稿；這些材料應先進入材料追蹤計畫。
- 不在資訊不足時硬補內容。

## 工作階段

### 多 agent 執行制度

本書可以由一個主控 agent 管控進度，再派多個章節 worker agent 逐講完成內容。

### 第 0 階段：建立書籍骨架
建立目錄結構、config (`configs/ee274-data-compression.yml`)、導讀，與 Lecture 1。

### 第 1 到 4 階段：逐字稿完整閱讀與寫作
每批次派發 worker，固定流程：
1. 開啟對應的逐字稿。
2. 從頭到尾完整閱讀。
3. 產出該講「抽象筆記」。
4. 改寫成繁體中文書稿章節。

### 第 5 階段：作業與課程資源整合
把 Slides、作業等材料以「關聯層」方式整合。

### 第 6 階段：外部補充與全書整合
補上網路補充、檢查 Mermaid 圖表、統一術語，並處理出版化準備 (MkDocs)。

## 品質檢查清單

- 每一章都能追溯到一份或多份完整讀完的逐字稿。
- 重要公式、系統圖、演算法或工程限制不可只用口語描述，必要時要補圖。
- 外部資料必須有引用紀錄。
- 全書術語要一致。
