# 第十四章：Exploration

> **逐字稿：** Lecture 14 Exploration（完整閱讀，2026-07-06）

## 導讀

為什麼 DQN 能輕鬆玩 Breakout，卻無法玩 Montezuma's Revenge？答案在於**探索**。本章從多臂拉霸機（bandit）建立理論基礎，再延伸到大規模 MDP 和 Meta RL 中的探索。

---

## 一、探索問題的本質

### 1.1 Montezuma's Revenge 為什麼難？

- 獎勵極稀疏（必須完成一長串動作才能得到獎勵）
- 智能體不理解「鑰匙」「門」「骷髏」的語義
- 前幾百步幾乎得不到任何回饋信號

**兩個對立目標：**
- **利用（Exploitation）**：做目前已知最好的動作
- **探索（Exploration）**：嘗試未知動作以期發現更好的策略

這個權衡就是**探索-利用困境**。

---

## 二、多臂拉霸機（Bandit）理論

### 2.1 問題形式

單步 MDP，無狀態，每輪選一個「臂」（action），獲得隨機獎勵。目標：最大化 $T$ 輪的累積獎勵。

### 2.2 後悔值（Regret）

$$\text{Regret}(T) = T \cdot \mathbb{E}[r(a^*)] - \sum_{t=1}^{T} r(a_t)$$

即：最優行動的期望總獎勵，減去實際得到的總獎勵。

| Regret 增長模式 | 代表意義 |
|---|---|
| 線性增長 | 算法沒有改進（每步都在犯同樣的錯）|
| 次線性增長 | 算法在學習，逐漸逼近最優 |
| 收斂至 0 斜率 | 探索完成，進入穩定利用 |

### 2.3 三種探索策略

**A. ε-greedy：**

以 $\epsilon$ 機率隨機選臂，$(1-\epsilon)$ 機率選已知最優臂。

- 優點：簡單
- 缺點：$\epsilon$ 固定時會持續浪費；$\epsilon$ 衰減需要調參

**B. UCB（Upper Confidence Bound，樂觀探索）：**

$$a_t = \arg\max_a \left[\hat{r}(a) + c\sqrt{\frac{\ln t}{N_t(a)}}\right]$$

- $\hat{r}(a)$：動作 $a$ 的歷史平均獎勵
- $\sqrt{\ln t / N_t(a)}$：不確定性估計（採樣次數少 → 不確定性高）
- 核心思想：「對未知的事物保持樂觀，直到真正嘗試後才更新」

**C. 後驗採樣（Thompson Sampling / Posterior Sampling）：**

對每個臂的獎勵概率維護信念分布（如 Beta 分布），每步：

```
1. 從每個臂的後驗分布採樣一個「期望獎勵估計」
2. 選擇採樣值最高的臂
3. 觀察真實獎勵，更新對應臂的後驗分布
```

**藥物開發游戲的實驗結果：**

| 算法 | 5000 個病人後的累積存活率 |
|---|---|
| 後驗採樣 | 最高 |
| UCB | 中等，長期優於 ε-greedy |
| ε-greedy | 初期較好，長期較差 |
| 純貪婪 | 差 |

## 三、大規模 MDP 的探索

### 3.1 為何大規模 MDP 探索困難？

從頭開始用隨機動作探索 Transformer 語言空間、機器人關節空間幾乎不可行——狀態空間過大，有效路徑無窮稀少。

### 3.2 實際做法

大型 RL 系統中，**探索主要依靠：**

1. **預訓練基礎模型**（演示資料或語言預訓練）：提供良好的探索起點
2. **示範資料**：引導策略進入有意義的狀態空間
3. **稠密（shaped）獎勵**：若可手工設計，能大幅引導探索方向

> 「隨機初始化的 Transformer 語言模型，直接用 RL 訓練幾乎不可能成功——現實中的 ChatGPT、Claude 都先有強大的預訓練模型作為探索起點。」

## 四、Meta RL 中的探索：DREAM 算法

### 4.1 重拾雞與蛋問題

（接第十三章）在有直接信息線索的環境中：

- 後驗採樣 = 逐條走廊試探（忽略牆上的標誌牌）→ 次優
- 端到端訓練 = 需要同時有好的探索和執行能力才能得到學習信號 → 優化困難

### 4.2 DREAM 核心思想

**解耦（Decouple）探索目標和執行目標：**

```
探索策略 π_explore：目標 = 收集能預測任務識別符 z 的資訊
執行策略 π_execute：目標 = 在已知 z 的條件下最大化獎勵
```

**獨立性：**
- 探索策略不依賴執行能力就能得到學習信號
- 執行策略不依賴好的探索就能得到學習信號（因為直接提供 $z$）

### 4.3 探索獎勵：資訊增益

設預測模型 $\hat{z}(\tau_{1:t})$ 是從觀測歷史預測任務識別符的模型，探索策略的逐步獎勵為：

$$r^{\text{explore}}_t = \|\hat{z}(\tau_{1:t-1}) - z\|^2 - \|\hat{z}(\tau_{1:t}) - z\|^2$$

即：此步驟讓我們的任務預測**減少了多少誤差**（資訊增益）。

### 4.4 完整 DREAM 算法

```
訓練（同時進行）：
A. 訓練 π_explore：最大化逐步資訊增益（預測 z 的準確率提升）
B. 訓練 π_execute：給定 z，最大化任務獎勵（多任務 RL）
C. 訓練預測模型 ẑ(τ)：最小化對 z 的預測誤差

測試：
1. 用 π_explore 探索新任務 → 收集軌跡 τ
2. 用 ẑ(τ) 推斷任務表示 ẑ
3. 用 π_execute(·|s, ẑ) 執行任務
```

### 4.5 壓縮任務表示（變分資訊瓶頸）

問題：若任務識別符 $\mu_i$ 是 one-hot，探索策略會嘗試預測包含無關資訊（牆的顏色）的識別符。

解法：學習壓縮的潛在表示 $z$，只保留對執行任務必要的資訊：

$$\mathcal{L} = \mathbb{E}[-\log\pi(a|s,z)] + \lambda \cdot D_{KL}[q(z|\mu) \,\|\, \mathcal{N}(0,I)]$$

加入高斯雜訊 + KL 正則化 → 迫使 $z$ 丟棄無關資訊。

探索策略的目標改為預測 $z$（壓縮表示），而非 $\mu$（原始識別符）。

### 4.6 樣本效率比較（理論）

設任務有 $A$ 個臂（選項）：

| 方法 | 達到最優所需樣本數 |
|---|---|
| 端到端訓練 | $O(A^2 \log A)$ |
| DREAM | $O(A \log A)$ |

DREAM 在問題規模增大時效率顯著更好。

### 4.7 實際應用：CS106A 作業評分

**任務：** 找出學生程式中的 bug（不同學生程式 = 不同任務）

- 探索策略：探索各種玩法找出 bug 觸發方式
- 執行策略：驗證是否成功觸發 bug
- **結果：** TAs 輔助評分速度提升 44%，準確率提升 6%

## 五、算法一覽：探索方法比較

| 方法 | 適用場景 | 優點 | 缺點 |
|---|---|---|---|
| ε-greedy | 小型 bandit | 簡單 | 長期效率差 |
| UCB | bandit、小型 MDP | 理論保證 | 高維難擴展 |
| 後驗採樣 | bandit、中型 MDP | 實驗效果好 | 有信息線索時次優 |
| 示範 + 預訓練 | 大型 MDP（LLM、機器人）| 實際可行 | 需要高品質示範 |
| DREAM | Meta RL | 最優+易優化 | 需任務識別符 |

---

## 小結

1. **探索-利用困境** 是 RL 的核心難題，稀疏獎勵讓它更嚴峻。
2. **Bandit 理論**提供了量化框架（後悔值）和有理論保證的算法（UCB、後驗採樣）。
3. **大規模 MDP** 中純隨機探索不可行；依賴預訓練模型和示範資料。
4. **Meta RL 的探索挑戰** = 雞與蛋問題：端到端最優但難優化，後驗採樣易優化但次優。
5. **DREAM** 解耦探索（資訊增益）和執行（任務獎勵），兼顧最優性和可優化性；樣本複雜度從 $O(A^2\log A)$ 降至 $O(A\log A)$。
