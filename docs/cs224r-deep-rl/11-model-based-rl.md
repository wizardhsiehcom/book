# 第十一章：Model-Based RL

> **逐字稿：** Lecture 11 Model-Based RL（完整閱讀，2026-07-06）

## 導讀

本章介紹一個基礎問題：若能預測「採取動作後世界會如何變化」，RL 訓練效率能提升多少？這被稱為**模型基礎強化學習（Model-Based RL, MBRL）**。核心構想是學習環境動態的模擬器，再在模擬器裡做規劃或生成訓練資料。

---

## 一、算法全景回顧（Lecture 11 開場）

在進入 MBRL 之前，本講先系統梳理課程至今介紹的全部算法類型：

```
線上 RL（Online RL）
├── 策略梯度（Policy Gradient，僅學策略）
│   └── REINFORCE（純在線，只用當前策略資料）
├── Actor-Critic（學策略 + 評論員）
│   ├── 有限 off-policy：PPO（重要性採樣 + Clip）
│   └── 完全 off-policy：SAC（Replay Buffer）
└── Q-Learning（僅學 Q 函數，隱式策略）
    └── DQN（完全 off-policy，Replay Buffer）

離線 RL（Offline RL，固定資料集，無環境互動）
├── 限制在資料內動作：AWR、AWAC、IQL
└── 保守 Q 值：CQL

模仿學習（Imitation Learning）
├── 離線：行為克隆（BC）
└── 在線：DAgger（在線詢問專家）
```

## 二、MBRL 的核心構想

### 2.1 什麼是動態模型？

動態模型（Dynamics Model）學習預測：

$$f_\phi(s_t, a_t) \approx s_{t+1}$$

即：**給定當前狀態 $s_t$ 和動作 $a_t$，預測下一個狀態 $s_{t+1}$**。

這等同於「學習一個模擬器」——不需要真實環境，只需模型預測的下一個狀態。

**現實中的動態模型類型：**

| 類型 | 例子 | 表示 |
|---|---|---|
| 已知模型 | 棋類遊戲（棋盤規則）| 直接使用，無需學習 |
| 部分已知 | 車輛行駛（知道運動學，不知道摩擦係數）| 擬合未知參數 |
| 完全未知 | 通用機器人操作、股市 | 從頭用神經網路學習 |

### 2.2 學習動態模型的方式

**目標：** 最小化預測下一狀態的誤差（L2 損失）：

$$\min_\phi \mathbb{E}_{(s,a,s') \sim \mathcal{D}}\big[\| f_\phi(s,a) - s' \|^2\big]$$

**觀測空間選擇：**

| 方法 | 在哪建模 | 適用場景 |
|---|---|---|
| 原始圖像空間 | 直接預測像素 | 高保真，計算量大 |
| 潛在空間（Latent Space）| 編碼器 $z=e(s)$，在 $z$ 空間預測 $z'$ | 丟棄無關細節，計算量小 |

**也需要獎勵模型：** 若獎勵無法解析表達（forward velocity 等），同樣需學習 $r_\phi(s,a)$。

## 三、MBRL 的基礎算法框架

```
初始化：收集一些初始資料 D（用隨機策略或初始策略）

循環：
  1. 在 D 上訓練動態模型 f_φ（最小化預測誤差）
  2. 用 f_φ 規劃：找出最優動作序列
  3. 在真實環境執行動作，收集新資料
  4. 將新資料加入 D，重回步驟 1
```

---

## 四、用動態模型規劃：三種方式

### 4.1 梯度反傳規劃（Gradient-Based Planning）

目標：找一個動作序列 $\hat{a}_{t:t+H}$ 最大化未來 $H$ 步的累積獎勵：

$$\max_{\hat{a}_{t:t+H}} \sum_{t'=t}^{t+H} r(s_{t'}, a_{t'})$$

**做法：** 既然動態模型和獎勵模型都是可微神經網路，可以對**動作本身**做梯度下降：

$$\hat{a} \leftarrow \hat{a} + \alpha \cdot \frac{\partial}{\partial \hat{a}}\sum_{t'} r(s_{t'}, a_{t'})$$

其中梯度透過鏈式法則從 $r$ → $s_{t'}$ → $f_\phi$ → $a_t$ 傳播。

**注意：** 這裡優化的不是神經網路參數 $\phi$，而是動作向量本身（連續動作空間）。

### 4.2 採樣基礎規劃（Sampling-Based / Zero-Order Optimization）

不用梯度，直接採樣多個動作序列，選最好的。

#### 隨機射擊法（Random Shooting）

```
1. 從初始分布採樣 N 個動作序列 A^(1), ..., A^(N)
2. 對每個序列滾動模型，計算累積獎勵
3. 選擇獎勵最高的序列
```

#### 交叉熵法（Cross Entropy Method, CEM）

迭代版隨機射擊，每輪更新採樣分布：

```
初始：從均勻分布採樣 N 個動作序列
迭代 K 次：
  1. 對每個序列計算累積獎勵
  2. 選出獎勵最高的前 M 個（精英集）
  3. 用精英集擬合一個高斯分布 N(μ, Σ)
  4. 從新分布重新採樣
返回：最後一輪中最佳序列的首個動作
```

**CEM 的優缺點：**

| 優點 | 缺點 |
|---|---|
| 天然可並行（多樣本同步評估）| 在高維空間難以覆蓋（維度災難）|
| 不需要可微模型 | 動作維度 × 規劃步長 H 都影響維度 |
| 適用離散動作 | — |

**梯度 vs 採樣比較：**

| | 梯度下降 | 採樣/零階 |
|---|---|---|
| 維度擴展性 | 好（高維亦可）| 差（維度災難）|
| 可並行性 | 差（必須序列計算）| 好 |
| 是否需要梯度 | 是 | 否 |
| 適用動作類型 | 連續 | 連續 + 離散 |

### 4.3 開環 vs 閉環規劃

**開環（Open-Loop）：** 規劃 $H$ 步動作序列，一次性全部執行，中間不接收反饋。

**閉環（Closed-Loop）：** 每步只執行首個規劃動作，觀察真實下一狀態，再重新規劃。

```
閉環規劃（MPC 模式）：
while not done:
  規劃 a_t, a_{t+1}, ..., a_{t+H}
  執行 a_t（只執行第一個）
  觀察真實 s_{t+1}
  t = t + 1
```

---

## 五、模型預測控制（Model Predictive Control, MPC）

MPC 是閉環規劃的正式名稱，是機器人和控制系統中廣泛使用的方法。

**核心優點：**
- 每步重新規劃，能對抗非確定性環境的變化
- 有效抵消模型誤差——即使當前模型不準確，下一步可以修正
- 把規劃步長 $H$ 限制在模型精確的短期範圍內

**暖啟動技巧（Warm-Starting）：** 用上一步的規劃初始化這一步，只隨機化最後一個動作，避免重複搜索。

**規劃頻率選擇：**

| 選項 | 優點 | 缺點 |
|---|---|---|
| 每步重新規劃 | 最準確 | 計算量大 |
| 每 $k$ 步重新規劃 | 計算效率更高 | 對快速變化環境反應慢 |

## 六、MBRL 的挑戰與解決方案

### 6.1 挑戰一：模型誤差累積

動態模型不完美，誤差在長序列中累積（多步預測後偏差指數增長）。

**解決方案：**
- 限制規劃步長 $H$（短視規劃 + 值函數補全長期）
- 閉環規劃（MPC）——每步從真實狀態出發
- 整合值函數：

$$V_{plan}(s_t) = \max_{a_{t:t+H}} \left[\sum_{t'=t}^{t+H} r(s_{t'}, a_{t'}) + V^\phi(s_{t+H})\right]$$

### 6.2 挑戰二：資料覆蓋不足

初始策略只訪問部分狀態，動態模型在未訪問區域不準確。

**解決方案：** 迭代資料收集——每次規劃後，把執行軌跡加回資料集重新訓練模型。

### 6.3 解決方案三：模型集成（Ensemble）

用 $K$ 個獨立神經網路（不同隨機種子、不同 mini-batch）建模集成，平均化單個模型的偏差：

```
訓練 K=3 個動態模型 f_φ^1, f_φ^2, f_φ^3
規劃時：同一動作序列在所有 3 個模型中評估
選擇在集成均值下獎勵最高的序列
```

這能避免「偏好某個模型的特殊漏洞」。

## 七、案例研究：靈巧手操作

### 7.1 問題設定

| 項目 | 設定 |
|---|---|
| 機器人 | 五指靈巧手（24 個關節，24 維動作空間）|
| 狀態 | 手部關節角度 + 物體 3D 位姿（運動捕捉）|
| 動作 | 連續關節角度控制 |
| 獎勵 | 目標軌跡跟蹤誤差 + 掉落懲罰 |
| 任務 | 保齡球旋轉（Baoding balls）、手寫數字 |

### 7.2 算法設計

- **動態模型：** 集成 3 個兩隱層（各 500 神經元）神經網路
- **規劃器：** 改良版 CEM（軟權重均值 + 時序平滑）
- **資料收集：** 每輪收集 30 條軌跡

### 7.3 結果與洞見

| 比較基準 | 樣本效率 | 最終性能（保齡球）|
|---|---|---|
| SAC（無模型）| ~500k 步 | 中等 |
| 自然策略梯度 | ~500k 步 | 中等 |
| 提出方法（MBRL + CEM 集成）| ~100k 步（**5× 更少**）| 最高 |
| 無集成（單模型）| ~100k 步 | 低於提出方法 |
| 僅隨機射擊（無 CEM 迭代）| ~100k 步 | 差 |

**關鍵發現：**
1. **集成必要**：沒有模型集成會因單模型誤差而性能下滑
2. **迭代採樣必要**：隨機射擊（無 CEM 迭代）遠不如 CEM
3. **規劃步長需權衡**：太短（未考慮長期後果）和太長（高維難優化）都差

**真實機器人結果：** 四小時真實資料 → 90° 旋轉成功率接近 100%；180° 旋轉也有顯著表現。

## 八、MBRL 的本質優勢與限制

### 優勢

- **樣本效率高**：相同性能，資料量通常比無模型 RL 少 5–10×
- **任務通用**：學到好的動態模型後，同一模型可用於不同獎勵（換目標不需重新採樣資料）
- **可解釋規劃**：每步動作有明確的「未來模擬」支撐

### 限制

- **測試時計算昂貴**：每步都要跑規劃（尤其 MPC + CEM），不能像策略網路只做一次前向傳播
- **模型質量瓶頸**：若模型精度差，規劃再好也無濟於事
- **短視問題**：僅適合稠密獎勵或中短期任務；稀疏長期獎勵需結合值函數

---

## 小結

1. **動態模型** 預測 $(s,a) \to s'$，學習方式從全知（棋類）到擬合（未知參數）到純神經網路。
2. **梯度規劃** 對連續動作高效，但依賴模型可微性；**CEM 採樣規劃** 可處理離散動作，天然可並行。
3. **閉環規劃（MPC）** 每步從真實狀態重新規劃，顯著抵消模型誤差，是機器人控制的標準做法。
4. **迭代資料收集** 解決覆蓋問題；**集成模型** 避免單模型漏洞。
5. 在樣本效率上，MBRL 比無模型方法好 5–10×；代價是更高的測試時計算量以及模型學習的額外複雜性。
