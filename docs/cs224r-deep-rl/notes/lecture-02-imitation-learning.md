## 本講主問題

給定專家示範資料 D = {(s,a)}，如何學出一個策略 π_θ 來模仿專家？本講討論三個子問題：(1) 版本零（行為複製）的本質及其失敗模式；(2) 如何用生成模型學習分布而非平均值；(3) 如何透過線上干預克服複合誤差。

## 重要細節

**多模態問題（開車示範）：**
- 直行策略 a ≈ 0，合流左策略 a ≈ -2
- 資料集分布：雙峰，均值 ≈ -1
- L2 迴歸輸出均值 ≈ -1，即跨線行駛，在資料分布下機率極低
- 結論：對多人示範資料，必須用表達能力夠強的分布

**分布類別對比：**
- 離散動作：直接 categorical → 最大表達能力，等同分類問題
- 連續：高斯 → 只能輸出均值（即使給 σ，也只會放寬高斯而非學到多模）
- GMM：事先選 N 個組件（超參數），輸出 N 組 (μ, σ, w)
- 自迴歸離散：每維離散化到 K 個 bin → 輸出 K 個機率值 → 用交叉熵；逐維條件預測（類語言模型）；維度順序可人工選或視為超參數（機械手臂可依運動鏈）
- Diffusion：表達能力最強，適合連續高維，但有迭代推理成本

**自迴歸訓練細節：**
- 訓練時輸入是資料集的真實動作（teacher forcing）
- 推理時輸入自身預測
- 損失 = 各維度交叉熵之和

**實驗結果（逐字稿）：**
- 搬運工具任務：Diffusion > GMM；多人資料集時差距更大
- 真實機器人（掛衣）：Diffusion vs L1，成功率差約 50%

**行為複製的離線特性：**
- 不需要獎勵函數
- 不需要環境互動
- 缺點：需要大量示範；Covariate shift 問題

**複合誤差機制：**
- 訓練分布 = 專家訪問的狀態
- 推理分布 = 策略訪問的狀態
- 策略出錯→落入稀少狀態→更大誤差→雪球

**DAgger 演算法：**
1. 展開當前策略 π_k
2. 對每個訪問到的狀態，查詢專家動作
3. 聚合到現有資料集
4. 在聚合資料集上更新策略
5. 重複
- 難點：「給定這個狀態，你應該怎麼做？」這個查詢在實際場景很難回答（例如方向盤角度）

**線上干預（實用版 DAgger）：**
- 讓策略跑，專家在看到錯誤趨勢時接管
- 記錄接管後的部分示範（不需要精確查詢）
- 自動駕駛安全員模型
- 缺點：反應可能太慢；某些災難性失誤來不及攔截
- 修正資料過窄（僅校內場景）→ 泛化性下降

## 工程限制與實務注意

- 表達能力強但計算昂貴（Diffusion 推理需多次去噪迭代）
- 多模態資料集（多人採集）一定要用表達能力強的分布
- 離散化 bin 的粒度是關鍵超參數：太粗 → 無法精確；太細 → 稀疏訊號
- 大型模型（Nvidia、Figure、Pi、Open-VLA、Waymo）均採用表達能力強的分布
