## 本講主問題

如何讓機器人在無人干預（無重置）的情況下，持續自主學習並改進？

## 核心概念

| 概念 | 說明 | 書稿章節 |
|---|---|---|
| 重置問題 | 機器人學習每次嘗試後需要人工重置環境 | 第一節 |
| 部署策略評估 | 訓練完成後策略從初始狀態的表現 | 第二節 |
| 終身獎勵評估 | 整個自主學習過程的累積獎勵 | 第二節 |

| Metal 算法 | 後向策略重置到示範狀態分佈（用判別器） | 第五節 |
| EARL 基準 | 每 200,000 步才允許一次重置 | 第五節 |
| 任務循環 | 根據當前狀態選擇最適合的任務練習 | 第六節 |

## 為何延長 Episode 行不通

1. **狀態漂移**：策略偏離高獎勵區域，稀疏獎勵無法引導回來
2. **狀態分佈崩塌**：replay buffer 集中在目標附近，忽略「從起點到目標」的狀態分佈

## Forward-Backward RL 算法

```
1. 初始化（僅一次）
2. 滾出前向策略 π_F（H步）→ 用 R_F 更新
3. 不重置，滾出後向策略 π_B（H步）→ 用 R_B 更新
4. 重複（偶爾才外部重置）
```

後向獎勵 $R_B$：到達初始狀態分佈

## Metal 算法改進

後向策略目標：不是到達初始狀態，而是到達**示範軌跡中的任意狀態**

- 用判別器 $D$ 區分策略狀態 vs 示範狀態
- $R_B(s) = -\log(1 - D(s))$（欺騙判別器）
- 等價於最小化策略分佈與示範分佈的散度

## 任務循環核心

- 學習多任務策略 $\pi(a|s, z_i)$
- 根據當前狀態選擇「可以練習的任務」（預定義圖或 LLM 提議）
- 解決前/後策略難度不對稱的問題

## Single Life RL 兩種策略

| 方法 | 機制 | 效果 |
|---|---|---|
| 熟悉狀態引導 | 獎勵「回到訓練時訪問過的狀態」| 四足機器人翻倒後自行站起 |
| 高層技能探索 | 在預訓練技能空間嘗試不同策略 | 解決需要「換策略」的新情境 |

## 與其他章節的比較

| 設定 | 描述 |
|---|---|
| 傳統 Episodic RL | 有頻繁重置，最後部署策略 |
| Forward-Backward RL | 無重置，自主訓練，部署前向策略 |
| Single Life RL | 有先驗訓練，一次部署，遇新情境自適應 |
