# 第二章：模仿學習

> **逐字稿：** Lecture 2 Imitation Learning（完整閱讀，2026-07-06）

## 導讀

這一章解決的問題是：在沒有明確獎勵函數的情況下，如何讓智能體習得有用的行為？答案是讓它觀察並模仿人類示範者。

模仿學習（Imitation Learning）假設我們擁有一批由專家 π* 採集的示範軌跡 D，目標是學出一個策略 π_θ，使它的行為盡可能接近專家。這聽起來簡單，但實際上有兩個隱藏的陷阱：其一，行為複製（Behavioral Cloning）的基本做法在面對多樣性示範時會失敗；其二，即使策略學得很好，部署時的「複合誤差」也會讓性能持續惡化。

---

## 一、行為複製：最樸素的做法

給定示範資料集 $\mathcal{D} = \{(s_t, a_t)\}$，最直觀的方法是訓練一個確定性策略 $\pi_\theta(s)$，讓它的輸出 $\hat{a}$ 盡量接近示範中的動作 $a$：

$$\min_\theta \sum_{(s,a) \in \mathcal{D}} \|\pi_\theta(s) - a\|^2$$

這等同於用監督學習做迴歸。在單一專家、行為一致的場景下，這樣就夠了。

### 失敗模式：多模態示範

考慮自動駕駛場景：有些示範者在某路段直行（$a \approx 0$），有些向左合流（$a \approx -2$）。資料集的動作分布是雙峰的：

```
機率
  │    ██                 ██
  │   ████               ████
  │  ██████             ██████
  └────────────────────────────→ 方向盤角度
       -2                  0
```

L2 迴歸最小化均方誤差，等同預測均值，結果落在 $a \approx -1$——兩個模式之間，在資料分布下機率極低，對應到在路面上橫跨兩條線的行為。

**核心洞察：** 分布的表達能力（expressiveness）是獨立於網路大小的。把神經網路做大，它仍然只能輸出一個均值，無法表示多模態分布。

---

## 二、學習動作分布而非均值

解法是讓策略輸出動作分布的**參數**，而不是單一動作值。

### 2.1 離散動作：Categorical 分布

對於離散動作空間（如遊戲按鍵），神經網路輸出每個動作的機率：$P(a_1), P(a_2), \ldots$。訓練目標是交叉熵損失，等同最大化示範動作的 log 概率。這是最大表達能力的選擇，因為它可以表示任何離散分布。

### 2.2 連續動作：從高斯到更複雜的分布

| 模型 | 輸出 | 表達能力 | 代價 |
|---|---|---|---|
| 單一高斯 | $(\mu, \sigma)$ | 低（只能表示高斯） | 最低 |
| 高斯混合模型 (GMM) | $N$ 組 $(\mu_k, \sigma_k, w_k)$ | 中（N 個高斯的組合） | 低，N 是超參數 |
| 自迴歸離散化 | 每維 $K$ 個 bin 機率 | 高（細粒度任意分布） | 中（需選 K 和維度順序） |
| 擴散模型 (Diffusion) | 迭代去噪步驟 | 最高 | 高（推理需多步） |

**訓練目標統一：** 無論哪種模型，目標都是最大化示範動作在策略分布下的 log 概率：

$$\min_\theta -\frac{1}{|\mathcal{D}|} \sum_{(s,a) \in \mathcal{D}} \log \pi_\theta(a \mid s)$$

### 自迴歸離散化詳解

以兩維動作（方向盤角度、油門）為例：

1. **離散化**：將每個維度分成 $K$ 個 bin
2. **逐維預測**：先輸出方向盤角度的分布（$K$ 個值的 softmax）；採樣 $\hat{a}_1$；將 $\hat{a}_1$ 輸入模型，輸出油門的條件分布
3. **訓練**：用 teacher forcing（輸入資料集中的真實動作），對每維計算交叉熵損失

這與語言模型逐 token 預測完全同構，可以直接借用序列模型（Transformer）實現。

**實驗驗證：** 在搬運工具任務（單人 vs 多人資料集）和機器人掛衣任務中，Diffusion > GMM > L2，且多人異質資料集時差距更大——表達能力越強，對多樣性越有容忍度。

## 三、複合誤差與分布偏移

### 3.1 為什麼行為複製在部署時會失效

行為複製是「離線」演算法——訓練完成後，策略在真實環境中執行，不再看到示範資料。部署時的問題在於：

```
訓練狀態分布 ≠ 部署狀態分布
       ↑
   Covariate shift
```

**複合誤差的雪球效應：**
```
策略小錯誤 → 偏離示範分布的狀態
           → 這些狀態在訓練資料中很稀少
           → 策略在稀少狀態下犯更大錯誤
           → 更遠離示範分布
           → ...
```

這與監督學習的關鍵差別是：監督學習的測試輸入和訓練輸入來自同一分布；模仿學習部署時，輸入是策略自己造成的，而策略一旦犯錯就改變了分布。

### 3.2 解法一：DAgger（資料集聚合）

**DAgger（Dataset Aggregation）** 是透過迭代收集修正資料來解決分布偏移：

```
迭代流程：
1. 展開當前策略 π_k → 收集訪問到的狀態序列
2. 對每個訪問到的狀態 s，查詢專家的正確動作 a*
3. 將 {(s, a*)} 加入資料集 D
4. 在 D 上訓練新策略 π_{k+1}
5. 重複
```

**效果：** 修正資料覆蓋了策略可能犯錯的狀態，讓策略學會如何「復原」。  
**難點：** 查詢步驟（「在這個狀態，你應該怎麼做？」）往往很難回答，尤其對連續動作（例如要求人明確說出方向盤角度）。

### 3.3 解法二：線上干預（實用版 DAgger）

不要求精確查詢，改讓專家在策略偏離時接管控制：

```
1. 啟動策略，讓它自主執行
2. 安全員（或監督者）觀察策略行為
3. 當策略開始偏離時，安全員接管並示範正確行為
4. 將這段「接管示範」加入資料集
5. 更新策略，重複
```

**優點：** 介面自然（安全員只需知道「什麼時候介入」，不需要精確回答查詢）。  
**缺點：** 反應可能比錯誤發生慢；修正資料範圍若過窄，策略泛化性下降。

## 四、模仿學習的適用場景

| 特性 | 說明 |
|---|---|
| 不需要獎勵函數 | 適合獎勵難以定義的任務（聊天、折衣、手術）|
| 不需要環境互動（BC） | 適合安全要求高的場景（醫療、核電）|
| 受限於示範品質 | 無法超越示範者（需要 RL 才能進步）|
| 需要大量示範 | 行為越複雜、越多樣，需要的示範越多 |
| 適合大規模預訓練 | 現代機器人基礎模型（Nvidia、Figure、Pi）的核心 |

---

## 小結

1. **行為複製 = 對示範動作的監督學習**，對單一一致的示範者效果好。
2. **L2 迴歸預測均值**，面對多模態示範時失敗——需要表達能力更強的分布。
3. **GMM、自迴歸離散化、Diffusion** 是三種可用的替代方案；表達能力越強，對多樣性資料越有用。
4. **Covariate shift** 讓部署時的分布不同於訓練分布；**複合誤差**使性能持續惡化。
5. **DAgger** 透過迭代查詢專家修正策略；**線上干預** 是其實用替代版本。
6. 模仿學習無法超越示範者——需要強化學習才能進一步改進。
