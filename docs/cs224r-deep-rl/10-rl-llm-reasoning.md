# 第十章：RL for LLM Reasoning

> **逐字稿：** Lecture 10 RL for LLM Reasoning（完整閱讀，2026-07-06）

## 導讀

讓 LLM 解決數學競賽題、寫代碼、做科學推理，是近年最引人注目的 AI 進展。這一章從 RL 角度分析：為什麼監督學習不夠、如何設計推理的 MDP、以及從 SFT 到 RFT 到在線 RL 的系統改進路線。

---

## 一、為什麼監督學習不夠？

**Supervised Fine-Tuning（SFT）的理論上界：**

$$\|P_\theta - P^*\| \leq O\!\left(D^{-0.15}\right)$$

學到的策略與「真實專家」的差距隨資料量 $D$ 縮小，但**指數極小**（$\approx D^{-0.15}$）。現有數據量到 2028 年前後就會耗盡，而模型距離真正的「研究員級別」推理能力仍然遙遠。

**SFT 的具體失敗模式：** 模型輸出「看起來合理但邏輯錯誤」的解答——每一步都像人類會寫的，但最終答案錯誤，且模型不會自我糾正。

---

## 二、推理任務的 MDP 框架

| RL 概念 | 推理中的對應 |
|---|---|
| 初始狀態 $s_0$ | 問題描述（tokens）|
| 動作 $a_i$ | 解答的第 $i$ 步（句子或段落）|
| 狀態 $s_i$ | $s_0 \| a_0 \| \ldots \| a_{i-1}$（歷史拼接）|
| 終止獎勵 | $+1$ 若最終答案正確，$0$ 否則（稀疏）|
| 動態 | 確定性（$s_{i+1} = s_i \| a_i$）|

這是一個**確定性動態、稀疏終止獎勵的 MDP**，不需要外部環境模擬器。

## 三、三條學習路線的比較

### 3.1 SFT：模仿人類解答

從資料集的（問題, 人類解答）對學習，純監督學習。

**缺點：** 資料昂貴；模型不能超越資料中已有的解題能力；測試誤差下降慢（$\approx D^{-0.15}$）。

### 3.2 RFT（Rejection Fine-Tuning）：在線過濾模仿

```
算法 RFT：
for 迭代 1 to T:
  1. 從當前策略 π_θ 對每個問題採樣 N 個解答
  2. 用答案比對過濾出正確解答集合 S̃
  3. 對 S̃ 做 SFT（最大化正確解答的 log 概率）
```

**優點：**
- 不需要人類撰寫解答，只需知道正確答案（用於過濾）
- On-policy 生成的解答更適合當前模型的學習
- **2倍資料效率**：等效測試誤差只需 SFT 一半的資料量

**問題：無限擴展會失效**

隨著 $|S̃|$ 增大，模型學到**虛假步驟（Spurious Steps）**：

```
示例：一個步驟算了 100×2=200（錯），
      下一步立即修正回 100÷2=50（對），
      最終答案正確。
模型學到這個錯誤步驟，卻仍然「過」了過濾。
```

這些虛假步驟在訓練問題上沒有影響，但在測試問題上導致泛化失敗——測試誤差在持續增加 $|S̃|$ 後反而上升。

### 3.3 在線 RL：直接策略梯度

不做過濾，直接用策略梯度在所有正確/錯誤解答上學習：

$$\nabla_\theta J \approx \sum_{y_i \sim \pi_\theta} R(y_i) \cdot \nabla_\theta \log\pi_\theta(y_i|x)$$

## 四、關鍵技術：每步信用分配

### 4.1 虛假步驟的根本原因

RFT 只在**軌跡級別**（整條解答）做過濾，無法分辨哪個步驟好、哪個步驟壞。需要**步驟級別**（Step-level）的評估。

### 4.2 Q 函數（過程獎勵模型，PRM）

對解答 $x; a_0, a_1, \ldots, a_{n-1}$ 的每個前綴定義 Q 值：

$$Q^{\tilde{\pi}}(s_i, a_i) = \mathbb{E}_{a_{i+1:\ldots}\sim\tilde{\pi}}\Big[\text{最終是否正確}\Big]$$

實現方法：**用一個「完成策略」$\tilde{\pi}$ 從當前前綴多次 rollout**，估計成功率。

**優勢函數（Advantage）：**

$$A(s_i, a_i) = Q(s_i, a_i) - Q(s_{i-1}, a_{i-1})$$

正優勢 → 步驟有益；負優勢 → 步驟有害（虛假步驟）。

### 4.3 優勢過濾（Advantage-Filtered SFT）

```
改進版 RFT：
1. 採樣解答軌跡
2. 在每個步驟做 rollout，計算 Q 值和優勢
3. 只在優勢 > 0 的步驟上做 SFT（即使整條軌跡是錯的）
4. 丟棄負優勢步驟（虛假步驟）
```

**效果：** 有效遏制虛假步驟的增長，測試誤差不再反升。

### 4.4 每步 DPO（Per-step DPO）

用 rollout 結果構建偏好對：

- 找到一個前綴，一個分支成功，另一個失敗
- 成功分支 → winning，失敗分支 → losing
- 構建偏好對，送入 DPO 訓練

**效果：** 相同資料量下，測試誤差比 SFT 低 **8倍**（資料效率提升 8×）。

## 五、在線 RL 方法

### 5.1 基礎：REINFORCE / 稀疏獎勵

直接在推理 MDP 上跑策略梯度，+1 獎勵（正確）或 0 獎勵（錯誤）。

### 5.2 GRPO（Group Relative Policy Optimization）

PPO 的變體，**不需要學習 V 函數**：

- 對同一個問題採樣一組解答
- 計算這一組解答的平均回報
- 用「相對於組內均值的偏差」作為優勢估計
- 加入 KL 約束防止偏離太遠

$$A_i = \frac{R_i - \text{mean}(R_1,\ldots,R_N)}{\text{std}(R_1,\ldots,R_N)}$$

### 5.3 在線 RL + 過程獎勵（PAV：Process Advantage Verifiers）

將過程獎勵模型（PRM）和在線 RL 結合：

1. 訓練 PRM 預測每個步驟的 Q 值/優勢
2. 在線 RL 訓練時，用 PRM 提供密集獎勵（每步都有信號）
3. 結果：**5-6× 樣本效率提升 + 6-7% 絕對性能提升**

## 六、各方法的資料效率比較

| 方法 | 相對 SFT 的資料效率 |
|---|---|
| SFT（人類解答）| 1× |
| RFT（on-policy 過濾）| ~2× |
| RFT + 優勢過濾 | 介中 |
| Per-step DPO | ~8× |
| 在線 RL + 過程獎勵 | 5-6×（+ 絕對性能提升）|

## 七、現代 Thinking Models（DeepSeek-R1、O-系列）

這些模型用的仍是以上同類 RL 算法（GRPO 等），關鍵差異在於：

**基礎模型更強**：能生成更豐富的「步驟」，包括：
- 解答後自我驗證
- 發現錯誤後回溯（backtracking）
- 嘗試多種解法

**更長的 token 預算**：RL 訓練時允許生成更長的解答序列，讓模型有更多「思考空間」。

**核心洞見：** 若基礎模型不具備自我驗證/回溯能力，RL 無法顯著提升——RL 是在已有能力的基礎上進行**選擇與強化**，而不是憑空創造新能力。

---

## 八、算法選擇建議

| 場景 | 建議方法 |
|---|---|
| 少量資料、需快速驗證 | RFT（rejection fine-tuning）|
| 中等資料、關注虛假步驟 | 優勢過濾 SFT |
| 豐富資料、強調效率 | Per-step DPO |
| 在線設置、追求最高性能 | GRPO + 過程獎勵 |

---

## 小結

1. **SFT 的數據瓶頸**：高品質推理資料快速耗盡，且模型被限制在人類示範的水準。
2. **RFT** 通過在線過濾提升效率（2×），但 scale up 後出現虛假步驟問題。
3. **步驟級 Q 函數** 實現細粒度信用分配；優勢過濾可清除虛假步驟。
4. **Per-step DPO** 在 offline 框架下達到 8× 資料效率。
5. **GRPO + 過程獎勵** 是目前在線 RL 中最強的訓練路線。
6. **Thinking models** 的成功 = 更強基礎模型（可自我驗證、回溯）+ 更長 token 預算 + 相同 RL 技術。
