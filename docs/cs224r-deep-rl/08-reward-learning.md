# 第八章：Reward Learning

> **逐字稿：** Lecture 8 Reward Learning（完整閱讀，2026-07-06）

## 導讀

前幾章都假設獎勵函數已知（遊戲分數、距離目標的負距離等）。但真實世界中，獎勵函數往往**難以明確定義**：

- 機器人倒水：水有沒有灑出？有沒有倒進杯裡？如何量化「做得好」？
- 對話系統：好的對話難以用數字直接衡量
- 自動駕駛：舒適度、規則遵守、安全性如何加權？

這一章討論如何**從人類監督中學習獎勵函數**。兩條路線：

1. **從示例目標學習**（Learning from Goals）：給成功/失敗的示例，訓練分類器
2. **從人類偏好學習**（Learning from Preferences，RLHF）：給人看軌跡配對，讓人選哪個更好

---

## 一、代理獎勵的侷限

在建立正式方法前，先看「直接設計代理獎勵」的問題：

| 代理獎勵 | 對應的真實目標 | 風險 |
|---|---|---|
| 點擊率 | 用戶滿意度 | 誘使點擊但無實際價值的內容 |
| 遊戲分數 | 玩遊戲的能力 | 過擬合到評分規則，而非真正「玩好」|
| 安全員接管次數 | 自動駕駛安全性 | 系統學會讓安全員忙，而不是真正安全 |

設計代理獎勵容易「reward hacking」——智能體找到讓代理獎勵高但真實目標差的行為。

---

## 二、從示例目標學習（Goal Classifier）

### 2.1 基本思路

1. 收集**成功狀態**（正例）和**失敗狀態**（負例）
2. 訓練**二元分類器** $f_\psi(s) \in [0,1]$，輸出成功機率
3. 用 $r(s) = f_\psi(s)$ 作為獎勵信號跑 RL

**問題一：** 獎勵稀疏（只有到達目標區域附近才有信號）。  
**問題二：** RL 算法會利用分類器的弱點——找到分類器認為是正例、但實際上不是成功的狀態（**reward hacking**）。

### 2.2 對抗性更新（Adversarial Training）修復 Reward Hacking

**關鍵洞見：** 把 RL 訪問的狀態加入**負例**並重新訓練分類器，讓分類器不斷學習策略的新行為模式。

```
算法（Goal Classifier with Adversarial Updates）：

1. 收集初始正例 S+ 和負例 S-
2. 訓練分類器 f_ψ on (S+, S-)
3. 用 r = f_ψ 跑 RL，收集策略訪問的狀態 S_rl
4. 將 S_rl 加入 S-（標記為負例）
5. 重新平衡資料集（正:負 = 50:50）
6. 返回步驟 2，迭代
```

**為什麼 50:50 平衡很重要？**  
若不平衡，隨著迭代負例越來越多，分類器可能學會「所有東西都是負例」。50:50 平衡確保即使部分加入的負例其實是「偽負例」（策略已學會到達目標），真正成功狀態的預測機率也能高於 0.5。

**與 GAN 的關係：** 這個算法本質上就是一個 GAN——策略是「生成器」，分類器是「判別器」，生成器試圖生成判別器認為是真實正例的狀態。

### 2.3 實驗結果

在機器人抓取任務中，以 50 條示範的最終狀態為正例：

| 方法 | 成功率 |
|---|---|
| 直接模仿學習（BC） | 26% |
| Goal Classifier + RL | 62% |

RL 利用獎勵信號，大幅超越純模仿學習。**關鍵：** 分類器必須有正則化，防止過擬合到 50 個正例。

## 三、從人類偏好學習（RLHF）

### 3.1 偏好比絕對評分更容易

問人類「這條軌跡的質量是 7.3 分嗎？」很難。  
問人類「A 和 B，哪個更好？」很容易。

**核心假設：** 人類的偏好 $\tau_W \succ \tau_L$（$W$ 優於 $L$）背後，是 $W$ 的累積獎勵更高：

$$\sum_{t} r(s_t^W, a_t^W) > \sum_{t} r(s_t^L, a_t^L)$$

### 3.2 Bradley-Terry 偏好模型

用 sigmoid 函數建模「$\tau_W$ 優於 $\tau_L$」的機率：

$$P(\tau_W \succ \tau_L \mid r_\psi) = \sigma\!\left(R_\psi(\tau_W) - R_\psi(\tau_L)\right)$$

其中 $R_\psi(\tau) = \sum_t r_\psi(s_t, a_t)$，$r_\psi$ 是用神經網路參數化的獎勵模型。

### 3.3 訓練獎勵模型

最大化人類偏好資料集的 log 似然：

$$\max_\psi \sum_{(\tau_W, \tau_L) \in \mathcal{D}} \log P(\tau_W \succ \tau_L \mid r_\psi)$$

等同最大化「高分軌跡的 $R$ 比低分軌跡高」這個損失。

### 3.4 完整算法

```
初始化：策略 π_θ，獎勵模型 r_ψ

迭代：
  1. 從 π_θ 採樣 K 條軌跡（建議同一 prompt/起點）
  2. 人類排序 K 條軌跡（得到 C(K,2) 對偏好標籤）
  3. 更新 r_ψ：最大化偏好對的 log 似然
  4. 用 r_ψ 作為獎勵，用 RL（如 PPO）更新 π_θ
  5. 重複
```

**為什麼要同一起點採樣 K 條？** 不同起點、不同難度的軌跡之間，很難比較相對好壞；同一起點讓比較更公平。

## 四、RLHF 在大語言模型中的應用

LLM 對齊的標準三段式流程：

```
階段一：大規模預訓練
  資料：海量混合質量文本
  目標：next-token prediction
  結果：知識豐富但行為難控的基礎模型

階段二：監督微調（SFT）
  資料：高質量示範（prompt → 理想回覆）
  目標：最大化 log π(ideal response | prompt)
  結果：可遵從指令的模型（本質上是 BC）

階段三：RLHF 微調
  3a. 採集偏好資料（人類標注哪個回覆更好）
  3b. 訓練獎勵模型 r_ψ(prompt, response)
  3c. 用 PPO 優化 π_θ 最大化 r_ψ，加 KL 約束防止偏離 SFT 模型太遠
  結果：更好地對齊人類偏好的模型（如 ChatGPT）
```

### RLHF 的 KL 約束

實務中 PPO 通常加入 KL 懲罰：

$$r_{total}(s,a) = r_\psi(s,a) - \beta \cdot \log\frac{\pi_\theta(a|s)}{\pi_{SFT}(a|s)}$$

防止模型為了最大化獎勵而偏離 SFT 模型過多（獎勵模型外推失真）。

## 五、RLAIF（AI 反饋代替人類反饋）

人類標注成本高、速度慢。替代方案：用一個**更強的 AI 模型**作為裁判：

- 輸入兩個回覆，詢問強模型「哪個更好？」
- 強模型輸出偏好標籤，代替人類標注
- 應用：內容安全（哪個回覆更無害）、幫助性評估等

**核心洞見：** 批評（判斷哪個更好）比生成（直接生成好的內容）更容易，因此弱模型可以從強模型的批評中學習。

---

## 六、兩種方法的比較

| | 從示例目標學習 | 從人類偏好學習（RLHF）|
|---|---|---|
| 監督形式 | 成功/失敗狀態示例 | 軌跡配對偏好 |
| 要求示範 | 是（成功示例）| 否 |
| 可擴展性 | 中（需要示例） | 高（LLM 規模）|
| 應用場景 | 機器人操控、目標達成 | LLM 對齊、複雜行為 |
| 主要挑戰 | 分類器被 reward hacking | 偏好資料採集成本 |

---

## 小結

1. **真實世界的獎勵函數難以直接設計**；代理獎勵容易被 reward hacking。
2. **Goal Classifier**：從成功/失敗示例訓練分類器作為獎勵，用對抗更新防止被利用，等價於 GAN。
3. **RLHF**：建模 Bradley-Terry 偏好概率，訓練獎勵模型，再用 PPO 優化策略——是 LLM 對齊的標準方法。
4. **偏好比絕對評分更容易標注**，且 $K$ 條軌跡排序可得 $C(K,2)$ 個偏好對，標注效率高。
5. **RLAIF** 以 AI 代替人類反饋，進一步降低標注成本。
