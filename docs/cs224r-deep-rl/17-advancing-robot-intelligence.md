# 第十七章：推進機器人智能

本章來自 2021–2022 年間的實驗性突破，展示了如何把強化學習從「在模擬中玩遊戲」推進到「在真實機器人上自主適應」。核心問題是 Sim-to-Real 轉移：如何讓模擬中訓練的策略在不微調的情況下，直接在真實世界的多變環境中工作。

## 17.1 模仿學習 vs. 強化學習：成功案例比較

理解強化學習在機器人上的定位，需要先比較兩者的成功形態：

| 維度 | 模仿學習 | 強化學習 |
|---|---|---|
| 資料來源 | Off-policy（人類示範、網路資料）| On-policy（自身 rollout）|
| 學習對象 | 正面示範 | 正面與負面資料均利用 |
| 輸出類型 | 通才模型（多任務）| 專才模型，或通才模型的精細化 |
| 成功案例 | 視覺語言模型、影片生成、圖像分割 | 圍棋/遊戲、LLM 推理、機器人靈巧操作 |

**兩者不是競爭關係**：AlphaGo 用 IL 預訓練，再用 RL 精進；LLM 用 SFT 初始化，再用 RLHF/GRPO 提升推理能力。機器人系統通常也先用 IL 取得基礎能力，再用 RL 突破人類示範的上限。

## 17.2 強化學習成功的兩個共同因素

觀察 AlphaGo（2016）、LLM 推理（2024）、機器人靈巧操作（2021）三個時代相差 7-8 年的成功案例，有兩個共同因素：

### 因素一：明確可驗證的獎勵

- **AlphaGo**：稀疏獎勵，但明確——贏了就是 +1，輸了就是 -1，圍棋規則完整定義了「贏」
- **LLM 推理（DeepSeek-R1）**：規則型獎勵函數，數學答案可自動驗證正確性
- **機器人**：追蹤目標速度，可從模擬器中精確計算

### 因素二：可擴展的大規模 rollout

- **AlphaGo**：圍棋規則完全在模擬中，算力直接轉化為資料
- **LLM**：在 GPU 集群上平行生成，reward model 自動評分
- **機器人**：關鍵突破在於「模擬器品質足夠好可以做 sim-to-real 轉移」

這兩個因素的交集，是機器人 RL 的瓶頸所在——模擬器不完美，獎勵難以設計，這正是本章要解決的問題。

## 17.3 Rapid Motor Adaptation（RMA）

**問題設定**：訓練一個四足機器人，能在各種地形（岩石、沙地、草地、泥地、油面）和各種條件（不同摩擦係數、隨機負重）下穩健行走，且不依賴視覺。

### Phase 1：帶特權資訊的教師策略

在模擬中，所有物理參數均已知，定義一個**外源向量**（extrinsics）$e$，包含：
- 地面摩擦係數
- 地面彈性係數
- 機器人質量偏移
- 各種物理擾動參數

訓練**教師策略**（base policy）$\pi_\theta(a | s, e)$：輸入觀測 $s$（關節角度、角速度、命令速度等）和特權外源向量 $e$，輸出關節扭矩。

**獎勵函數設計**（10個分量）：

| 類型 | 分量 | 目的 |
|---|---|---|
| 任務 | 追蹤目標線速度 | 核心任務 |
| 任務 | 追蹤目標角速度 | 轉彎能力 |
| 能量 | 最小化關節扭矩 | 節能 |
| 能量 | 最小化關節加速度 | 平滑動作 |
| 安全 | 懲罰地面衝擊力 | 保護硬體 |
| 穩定 | 懲罰機身偏轉 | 抗擾動 |
| 穩定 | ... | ... |

用 PPO 在模擬中訓練，消耗約 **10 億個採樣**收斂。

### Phase 2：學生策略的知識蒸餾

**問題**：真實機器人不知道外源向量 $e$（摩擦係數不可直接量測）。

**解法**：訓練一個**適應模組**（adaptation module），從觀測歷史 $\tau_{1:t} = \{(s_1, a_1), ..., (s_t, a_t)\}$ 中估計 $e$：

$$\hat{e}_t = f_\phi(\tau_{1:t})$$

**為何觀測歷史包含 $e$ 的資訊？**

若地面特別滑，機器人發出「向前移動0.1m」的指令，但實際位移只有 0.05m——這個「指令與結果的差異」正是地面摩擦的信號。觀測歷史中的誤差模式隱含了環境的物理特性。

**DAgger 蒸餾**：

1. 初始化適應模組（隨機權重）
2. 用當前適應模組 rollout，生成軌跡
3. 用教師模組在同樣軌跡上標記每一步的「真實 $e$」
4. 用 supervised learning 更新適應模組：$\min_\phi \|\hat{e}_t - e_t\|^2$
5. 重複直到收斂

**部署架構**：

```
觀測歷史 τ → 適應模組 f_φ（10 Hz）→ ê
當前觀測 s  ┘
                → 基礎策略 π_θ（100 Hz）→ 關節扭矩
```

適應模組只需 10Hz（受限於板載算力），基礎策略 100Hz——這個非同步並未在訓練中建模，但實測並未影響性能。

### Phase 1 vs Phase 2 比較

| | Phase 1 教師策略 | Phase 2 學生策略 |
|---|---|---|
| 輸入 | $s$, 真實 $e$ | $s$, 歷史 $\tau$ |
| 訓練方式 | PPO（on-policy RL）| DAgger（supervised）|
| 可部署？ | 否（需要特權資訊）| 是 |

## 17.4 RMA 盲行走結果

**完全不使用視覺**，只靠本體感知（IMU、關節編碼器、命令速度）：

- **岩石、碎石、泥地**：腳踩不穩時適應模組偵測到扭矩異常，自動增加抓地力
- **油面（塑膠布）**：滑動時識別出摩擦下降，調整步態
- **5 kg 負重**（機器人自重 12 kg）：偵測到重心前移，增加前腳扭矩
- **泡沫地墊**：下陷時識別出阻力特性，調高膝蓋清除高度

**適應模組消融實驗**：關掉適應模組（固定第一步估計的 $e$），在 8kg 負重下機器人失去平衡前傾。加上適應模組後自動修正。

**基準比較**：

| 方法 | 特點 | 結果 |
|---|---|---|
| Domain Randomization（無 $e$ 輸入）| 盲目魯棒 | 扭矩高、顛簸 |
| System ID（直接估計物理參數）| 試圖精確估計摩擦等 | 低於魯棒策略 |
| Fine-tune at inference（在真實世界更新）| 慢、樣本效率低 | 中等 |
| RMA | 歷史推斷 + DAgger | 最佳 |

## 17.5 視覺感知整合：不用地圖

機器人在某些場景**必須**使用視覺：跨越間距（需要精確落腳點）、減少硬碰到物體（優雅 + 硬體壽命）。

**傳統方法**：感知 → 建立地形地圖 → 控制器。問題：地圖是有損壓縮，小誤差在複雜地形（精確間距的墊腳石）會導致大失敗。

**本工作**：**直接把深度圖像輸入策略**，省去中間的地圖。

### Phase 1：用地形高度圖訓練教師策略

在模擬中，完美地形高度圖已知。訓練教師策略 $\pi(a | s, h_{\text{map}})$，其中 $h_{\text{map}}$ 是以機器人為中心的高度圖網格。不添加噪聲，速度快（無需渲染深度圖）。

### Phase 2：用深度圖蒸餾

訓練深度圖編碼器：$f_\phi(\text{depth image}) \approx$ 教師在相同軌跡上用到的「地形特徵」。用 DAgger：

- 學生策略用深度圖 rollout
- 教師策略用高度圖提供監督標籤
- 同時也蒸餾外源向量 $\hat{e}$

部署時只需深度相機 + 本體感知，無需 GPS、LiDAR、地圖建構。

### 實測結果

使用 RNN 處理深度歷史（保留短期上下文）：

- 墊腳石（跨越間距）：成功穿越，相比地圖方法在困難地形更準確
- 不均勻樓梯：小型機器人相對腿長較短，學到「側向伸腿」的新步態
- 碎石坡、泥濘、滑面：重試後成功（in-context 學習）

**基準比較**（困難地形）：

| 方法 | 墊腳石成功率 |
|---|---|
| 盲行走 | 低 |
| 地圖 + 控制 | 低（地圖噪聲影響大）|
| 直接深度 | 最高 |

有趣發現：在樓梯地形上，**盲行走比地圖方式更好**——地圖的噪聲使控制器不如直接學習盲步態。

## 17.6 靈巧操作：手中旋轉物體

**同樣的 RMA 框架**可應用於 5 指機械手（24 個關節）的靈巧操作。目標：在手中旋轉任意物體到指定姿態。

- **零樣本泛化**：訓練時用模擬的球形物體，測試時用：乒乓球、紙捲、橘子、冰球（光滑）、奇異果（粗糙）、方塊、杯子
- **物理屬性跨度**：重量 5g 到 200g，摩擦係數、質心位置、形狀均不同
- **推論**：本體感知歷史讓策略學到接觸偵測能力——「關節命令與實際位置的差距」揭示接觸力，從而推斷物體大小和形狀

## 17.7 無人機多形態適應

同一套策略權重控制**不同規格的無人機**（臂長差 3 倍、質量差 4 倍），通過觀測歷史動態推斷自身形態。部署初期短暫不穩，幾秒後識別出形態，飛行平穩——甚至在螺旋槳被網球擊中後維持飛行。

## 17.8 現狀與展望

### Tesla Optimus 進展

| 能力 | 現狀 |
|---|---|
| 雙足行走 | 工作，零樣本部署 |
| 舞蹈動作 | 工作，實時板載 |
| 語言條件操作 | 部分工作，但不可靠 |
| 精細靈巧操作 | 尚未達到可靠程度 |

操作系統使用「人類 egocentric 影片 + 少量機器人資料」訓練，語言條件化——概念正確，但可靠性不足。

### 為何操作仍難？兩個瓶頸

**瓶頸一：模擬品質**

| 任務 | 模擬可行性 |
|---|---|
| 剛體接觸（行走）| 已成熟，Sim-to-Real 可行 |
| 可變形物體（揉麵）| 可模擬但速度慢 |
| 複雜流體（切蔬菜汁液）| 幾乎不可模擬 |

「苦澀教訓」（Rich Sutton）：歷史上依靠算力的通用方法勝過領域特定方法。隨著算力增加，模擬品質會逐步提升——這是樂觀的理由。

**瓶頸二：獎勵設計**

行走只需一個獎勵函數（追蹤速度），可跨所有地形泛化。操作的每個任務需要不同的獎勵——「切蔬菜」的獎勵無法用於「揉麵」。

解法候選：訓練一個**通用獎勵模型**（VLM/VLM-R），理解人類意圖並評估任何任務的執行品質。挑戰：RL 會利用獎勵模型的弱點。絕對最壞情況：人工標注所有獎勵——人類有 80 億個，可規模化。

### 超越人類能力

AlphaGo 的教訓：稀疏獎勵 + 大規模搜索 → 發現人類從未想到的圍棋招式。機器人不受碳基生物的速度、耐久、精度限制，理論上可達到人類無法企及的能力。**達到人類能力水平，不是終點，而是起點。**

## 17.9 章節總結

```
Sim-to-Real 的核心解法

Phase 1：帶特權資訊訓練教師策略（模擬中）
          ↓ DAgger 知識蒸餾
Phase 2：從觀測歷史估計環境外源
          ↓ 零樣本部署
真實世界：在線適應，無需微調
```

成功的必要條件：明確獎勵 × 高品質模擬 × 在線適應機制。
