# 第十五章：Hierarchical RL and IL

> **逐字稿：** Lecture 15 Hierarchical RL and IL（完整閱讀，2026-07-06）

## 導讀

清理一間臥室、預訂一週的假期行程、修復造成神經網路損失爆炸的 bug——這些都是「長視野任務」。它們有多個相互依賴的子步驟，需要做出許多決策。本章介紹用層級結構（Hierarchy）來攻克這類任務。

---

## 一、長視野任務的困難

### 1.1 為什麼難？

| 困難 | 說明 |
|---|---|
| 狀態空間廣 | 執行途中訪問大量不同狀態，策略需要在所有狀態上都表現良好 |
| 錯誤機會多 | 長序列中任何一步出錯就可能導致任務失敗 |
| 容易卡住 | 若某個錯誤讓策略回到同一狀態，策略會無限重複同一錯誤 |
| 稀疏獎勵 | 只有最終完成才有獎勵，探索極其困難 |

### 1.2 層級的核心思想

與其讓一個策略直接輸出底層動作，不如：

```
高層策略（High-Level Policy）π_HL：
  輸入：當前狀態 + 整體目標
  輸出：下一個子目標（Goal）→ 給低層策略執行

低層策略（Low-Level Policy）π_LL：
  輸入：當前狀態 + 子目標
  輸出：底層動作（關節力矩、token 等）
```

**類比：** 做烤芝士蛋糕
- 整體任務 → 購買食材 → 去超市 → 走到門口 → 邁步 → 收縮肌肉

高層策略負責規劃「買食材」，低層策略負責執行「邁步」。

---

## 二、為什麼層級有幫助？

| 益處 | 說明 |
|---|---|
| 中間監督信號 | 低層策略對完成子目標有明確獎勵，不需要等到整個任務完成 |
| 跨子任務知識共享 | 低層策略以子目標為條件，相似子目標可共享權重 |
| 結構化探索 | 高層策略在「子目標空間」探索，而非原始動作空間 |
| 計算效率 | 高層規劃可在低層的低頻時間點執行（例如：每 50 步規劃一次）|

> **重要警告：** 若高層和低層策略完全共同訓練且無中間監督，本質上等同於一個「扁平策略」加上特殊架構——不一定有效。層級帶來的好處依賴於顯式的中間監督設計。

---

## 三、設計選擇 I：子目標的表示形式

子目標（Skill / Option / Subgoal / High-Level Action）的表示決定了低層策略的能力上限。

| 表示形式 | 適合場景 | 優點 | 缺點 |
|---|---|---|---|
| 語言字串 | 機器人操作、LLM 智能體 | 表達力強，可泛化 | 需要語言標注 |
| 目標圖像 | 視覺機器人任務 | 無需語言標注 | 需要生成模型 |
| GPS 座標 | 自駕、機器人導航 | 結構清晰（兩點越近越相似）| 不通用 |
| 狀態向量 | 連續控制 | 精確 | 難泛化 |

**好的子目標表示應具備：**
1. **表達力**：能描述多種不同的低層行為
2. **結構性**：相似目標有相似表示（GPS 的優點）
3. **合適的抽象層次**：不太細（預測每個像素）也不太粗（「去廚房」不夠細化）

---

## 四、設計選擇 II：各層的監督

### 4.1 低層策略的監督

**目標：** 到達子目標，不是完成整體任務

$$r_{LL}(s, a, g) = -d(s', g) \quad \text{或} \quad \mathbb{1}[d(s',g) < \epsilon]$$

### 4.2 高層策略的監督

**目標：** 預測子目標，使整體任務完成

$$r_{HL}(s, g^{HL}) = r_{\text{整體任務}}(s, a)$$

### 4.3 兩層需要協調

**問題：** 若分開獨立訓練，低層策略「找食材」的訓練分布可能和高層策略實際調用時的狀態分布不匹配。

**解法：** 至少讓一個策略適應另一個；最理想是聯合微調。

**LLM 的角色：** 大型語言模型可直接生成合理的高層計劃（「取出鍋」→「裝水」→「煮沸」），但需要微調以適應低層策略的實際能力。

---

## 五、設計選擇 III：何時請求新子目標？

### 方案 A：低層完成後再請求

```
if 低層策略判斷子目標已完成:
  請求高層策略給出下一個子目標
```

**優點：** 計算效率最高  
**缺點：** 判斷「是否完成」本身很難；若判斷錯誤（認為沒完成）會讓策略卡住

### 方案 B：固定步數後請求（推薦）

```
每 N 個低層 time-step 請求一次新子目標
```

**優點：** 簡單，高層策略更頻繁，錯誤影響更小  
**缺點：** 可能在子目標執行中途中斷；N 的選擇需要權衡

**實踐共識：** 大多數系統用固定步數方案，因為完成判斷的錯誤更危險（讓策略永久卡住）。

## 六、案例一：語言子目標的層級 IL

### 6.1 資料準備

1. 收集完整長視野示範軌跡（例：製作 trail mix）
2. 將軌跡分段，每段加上語言標籤（例：「拿起勺子」、「舀花生」、「倒入袋子」）
3. 分段長度可以不固定（實際系統用固定頻率重新查詢）

### 6.2 訓練

| 策略 | 輸入 | 輸出 | 訓練方式 |
|---|---|---|---|
| 高層策略（~1 Hz）| RGB 圖像 | 語言子目標字串 | 行為克隆 |
| 低層策略（50 Hz）| RGB 圖像 + 關節角度 + 語言子目標 | 關節命令 | 行為克隆 |

### 6.3 DAgger 在高層空間的應用

- 凍結低層策略，用 DAgger 微調高層策略
- 人類在語言空間發出修正命令（例如：「往左移」「往高一點」）
- 高層策略學會補償低層策略的缺陷

**量化結果（類似系統的比較）：**
- 層級 vs 扁平策略：任務完成進度提升 **34%**
- DAgger vs 純 IL（高層策略）：指令準確率有顯著提升

### 6.4 實際效果

單一套系統能完成「清理臥室」（撿起衣服、調整枕頭、扔垃圾）——傳統扁平策略很難做到。

## 七、案例二：圖像子目標的層級 IL

### 7.1 構想

高層策略用**擴散模型**生成「未來目標圖像」，低層策略以（當前圖像, 目標圖像）為條件輸出動作。

```
高層：(觀測圖像, 整體目標) → 生成模型 → 子目標圖像（粉色邊框）
低層：(觀測圖像, 子目標圖像) → 行動序列（逆模型）
```

### 7.2 優點

- **不需要語言標注**：只需視頻示範，就能學習高層策略
- **可使用人類示範視頻**（YouTube）訓練高層策略，擴充資料規模
- **對未見過的物體更有彈性**：不需要知道語義名稱

### 7.3 效果

- 加入人類視頻訓練高層策略 → 成功率顯著高於只用機器人資料

## 八、層級強化學習

### 8.1 狀態目標條件策略 + 事後重標籤

- 低層策略訓練：到達目標狀態（相對位移）為獎勵
- 高層策略訓練：輸出目標狀態作為高層動作（off-policy，使用 Replay Buffer）
- 事後重標籤：把高層動作（目標狀態）重標籤為實際到達的狀態

**實驗結果：** 機器人需要推開障礙物才能到達目標的任務，只有層級策略成功。

### 8.2 技能發現（Skill Discovery）

**目標：** 在無監督條件下自動發現有用的技能

- 半獵豹（Half Cheetah）學到：向前跑、向後退、前翻筋斗
- 原理：最大化技能之間的可區分性（互資訊）
- **現狀**：仍是活躍研究方向，在大規模狀態空間尚未成熟

## 九、Chain-of-Thought 作為層級的替代

現代 LLM 中的 Chain-of-Thought（思維鏈）可視為「內化的層級」：

```
輸入任務 → 先輸出中間步驟（子目標）→ 再輸出最終動作
```

**vs. 顯式層級：**
- 優點：合為單一模型，不需要分開訓練
- 缺點：每個 time-step 都需要完整的推理計算（高層計算不能在低頻執行）
- 研究上：尚無定論哪個更好

---

## 小結

1. **長視野任務** 因稀疏獎勵、廣大狀態空間和錯誤累積而極難。
2. **層級** = 高層策略預測子目標，低層策略執行；關鍵在於**顯式的中間監督**。
3. **子目標表示** 應具備表達力、結構性和合適的抽象層次；常見有語言、圖像、座標。
4. **各層監督**：低層訓練完成子目標獎勵，高層訓練完成整體任務獎勵；兩層需協調。
5. **重新查詢頻率**：固定步數（方案 B）比完成判斷（方案 A）更穩健，普遍被採用。
6. **層級 IL** 用分段示範 + DAgger；**層級 RL** 結合 Hindsight Relabeling 和 off-policy 算法。
7. **技能發現** 是無監督學習子技能的方向，大規模場景尚待突破。
