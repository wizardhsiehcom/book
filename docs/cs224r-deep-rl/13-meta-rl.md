# 第十三章：Meta RL

> **逐字稿：** Lecture 13 Meta RL（完整閱讀，2026-07-06）

## 導讀

強化學習系統學一個新任務需要大量樣本，但人類可以在幾分鐘內學會使用從未見過的咖啡機。差距在哪裡？人類有**先驗經驗**。Meta RL 的目標：訓練出能利用少量新任務資料快速適應的策略。

---

## 一、問題定位

### 1.1 三條路線的比較

| 方法 | 原理 | 樣本效率 | 所需訓練任務 |
|---|---|---|---|
| Fine-Tuning（微調）| 在源任務上訓練，在目標任務上微調 | 低（需要大量新任務資料）| 1 |
| Multi-Task Learning | 同時在多任務上訓練，zero-shot 泛化 | 中（需要高度相似的任務描述）| 多個 |
| Meta RL | 顯式優化「用少量資料快速適應」的能力 | 高（few-shot 或 in-context）| 多個（用於 meta-training）|

### 1.2 Meta RL 的形式定義

- **訓練分布：** 一組 MDP $\{M_i\}$，每個有自己的動態和獎勵
- **目標：** 給定新任務 $M_j$（在訓練分布內）的 $K$ 條 episode 資料，快速學出高回報策略
- **假設：** 新任務在訓練分布內（不需要完全一樣，但不能完全不同）

---

## 二、Black-Box Meta RL：用記憶體做 In-Context 適應

### 2.1 核心想法

訓練一個**帶記憶的策略**（序列模型），讓它把過去的經驗（狀態、動作、獎勵）作為 context，在 context 內自動完成探索→利用的過程。

```
訓練算法：
for 每輪迭代:
  1. 採樣任務 M_i
  2. 在 M_i 裡滾動策略 N 個 episode（保持記憶跨 episode）
  3. （可選）把 N 個 episode 存入 Replay Buffer
  4. 最大化所有任務的期望累積獎勵
     max_π E_{i ~ tasks}[∑_t r(s_t, a_t)]  ← 記憶跨 episode
```

**與標準 RL 的關鍵區別：**

| 項目 | 標準 RL | Black-Box Meta RL |
|---|---|---|
| 記憶 | 通常無跨 episode 記憶 | 記憶跨 N 個 episode |
| 獎勵輸入 | 只在訓練用 | **測試時也作為輸入**（用於任務識別）|
| 訓練域 | 單一 MDP | 跨多個不同 MDP |
| 探索 | 外部控制（ε-greedy 等）| 由記憶自動學習探索策略 |

### 2.2 架構選擇

| 架構 | 特點 |
|---|---|
| RNN（LSTM）| 最早期，相對較弱 |
| Self-Attention + 1D 卷積 | 更好，近似 Transformer |
| Transformer | 現代標準，最強 |
| Deep Set（前饋 + 均值）| 快速、可並行、驚人地有效 |

### 2.3 測試流程

```
測試時：
1. 在新任務 M_j 中用策略探索（最多 N 個 episode）
2. 策略的記憶不斷累積
3. 策略自動切換到利用模式（因訓練目標是最大化 N 個 episode 的總獎勵）
```

## 三、案例：視覺迷宮導航

- **設定：** 測試時用第一人稱視角，在從未見過的迷宮中找目標
- **N = 2 個 episode：** 第一個 episode 探索，第二個 episode 直奔目標
- **效果：** 第二個 episode 的找目標時間遠短於隨機策略

**量化結果：** Self-Attention + 卷積架構 > LSTM；第二個 episode 顯著比第一個 episode 快。

---

## 四、Meta RL 與測試時計算（Test-Time Compute）

**DeepSeek R1 / O 系列的類比：**

- 難題 → 模型「思考」更長
- 簡單題 → 快速給出答案
- 核心 = Meta RL：用不同難度的題目訓練，讓模型學會**動態分配探索計算量**

**效果：** 相同 token 預算下更高準確率；相同準確率下更少 token。

## 五、Meta RL 的探索挑戰

### 5.1 雞與蛋問題

端到端訓練理論上可學到最優的探索-利用平衡，但有致命難題：

```
場景：有走廊的環境，只有找到獎勵後才能學到「如何探索」
問題：
  - 沒有探索 → 沒有獎勵 → 沒有學習信號 → 不會探索
  - 沒有任務執行能力 → 即使探索也得低獎勵
→ 陷入「雞與蛋」的耦合死鎖
```

### 5.2 三條替代路線

**A. 後驗採樣（Thompson Sampling / Posterior Sampling）：**

維護對任務 $z$ 的信念分布，採樣任務假設並按其行動：

```
1. 從先驗採樣 z ~ p(z)
2. 按 z 對應的策略行動，收集資料
3. 更新後驗 p(z|data)
4. 重複（信念越來越窄，行動越來越集中）
```

**適合：** 明確試探不同假設的場景。
**不適合：** 當「正確信息」在環境中某處可直接獲取（走廊盡頭有標誌牌），後驗採樣會浪費時間逐條走廊試探而不去讀標誌牌。

**B. 內在獎勵（Intrinsic Motivation）：**

給新狀態加探索獎勵，鼓勵訪問未見過的狀態——可能探索不相關細節（牆上的畫）。

**C. DREAM（解耦探索-利用）：** 見第十四章詳述。

## 六、部分可觀測 MDP（POMDP）視角

**多任務 RL** 中任務識別符 $z_i$ 已知，是完全可觀測的聚合 MDP。

**Meta RL** 中 $z_i$ 未知，策略必須從觀測推斷，本質是 **POMDP**：

$$\text{觀測} = (s, R, \text{history}), \quad \text{隱藏狀態} = z_i$$

用記憶體推斷 $z_i$ = 用觀測歷史解決 POMDP。

## 七、潛在任務表示（Latent Task Representation）

### 7.1 從任務識別符到潛在向量

問題：One-hot 任務索引對新任務無法泛化。
解法：學習連續的低維任務表示 $z \in \mathbb{R}^d$，讓相似任務的 $z$ 也相近。

### 7.2 變分資訊瓶頸（Variational Information Bottleneck）

為了讓 $z$ 只保留**任務相關**資訊（丟棄無關細節如牆的顏色）：

$$\min_\phi \mathbb{E}\big[-\log\pi(a|s,z)\big] + \lambda \cdot D_{KL}\big[q_\phi(z|\mu_i) \,\|\, \mathcal{N}(0,I)\big]$$

- 第一項：最大化獎勵（解任務）
- 第二項：KL 懲罰壓縮 $z$，去除無關資訊（等效於加高斯雜訊 + 限制 $z$ 的模長）

---

## 小結

1. **Meta RL 目標** = 訓練策略能以少量新任務資料（K-shot）快速適應。
2. **Black-Box 方法** = 訓練有跨 episode 記憶的序列模型，端到端優化多任務累積獎勵。
3. **測試時**：策略用記憶自動完成探索→利用的切換，無需外部干預。
4. **探索挑戰** = 雞與蛋問題：若探索和執行技能同時欠缺，無法獲得學習信號。
5. **後驗採樣** 是相對易優化的替代方案，但對「有直接信息線索」的任務是次優的。
6. **DREAM** 解耦探索（預測任務 ID）和執行（最大化任務獎勵），同時達到最優和易優化。
