# 第十六章：機器人的自主強化學習

機器人強化學習有一個被忽視的假設：每次嘗試失敗後，有人會把環境重置回起點。真實機器人系統不能呼叫 `env.reset()`，而人力重置讓自主學習成為空話。本章探討如何設計能在無人干預下持續改進的機器人學習算法。

## 16.1 重置問題

傳統 RL 訓練循環是：從初始狀態分佈 $\rho_0$ 採樣 → 滾出策略 → 收集資料 → 更新策略 → **重置** → 重複。在模擬中，`env.reset()` 是免費的。在真實機器人上，這個步驟可能需要：

- 人工把機械臂移回初始位置
- 人工把散落的物品擺回桌上
- 人工把機器人從死角抱回起點

如 Lecture 16 所示，研究人員花大量時間當「重置工」——不斷把毛巾展開讓機器人再次嘗試折疊，或把門關上讓機器人再次嘗試開門。RL 通常需要數千次嘗試，這些人力干預讓學習既昂貴又緩慢。

**目標**：設計一個初始化一次、之後讓機器人完全自主練習的學習系統。

## 16.2 兩種評估視角

自主學習系統有兩種本質上不同的評估方式：

### 部署策略品質（Deployed Policy Evaluation）

學習結束後，部署學到的策略，從初始狀態分佈出發衡量其表現：

$$J_{\text{deploy}}(\pi) = \mathbb{E}_{s_0 \sim \rho_0}\left[\sum_{t=0}^H r(s_t, a_t)\right]$$

**適用場景**：訓練期的笨拙過程不重要，只關心最終部署品質——例如訓練一個上菜機器人，顧客不在乎它訓練時打翻了幾次盤子。

### 終身獎勵（Lifetime Reward）

整個學習過程中累積的獎勵，以平均獎勵在時間趨於無窮時的極限衡量：

$$J_{\text{life}}(\pi) = \lim_{H\to\infty} \frac{1}{H}\mathbb{E}\left[\sum_{t=1}^H r(s_t, a_t)\right]$$

**適用場景**：火星探測車部署後無法召回，訓練期間的每次行動都有實際意義。

## 16.3 為什麼單純延長 Episode 行不通

最直覺的想法：把 episode 長度從 $H=1000$ 拉到 $H=100000$，自然就不需要那麼頻繁重置。實驗結果顯示這幾乎無效：

**問題一：狀態漂移**
策略在學習初期會「漂移」出高獎勵區域。一旦偏離，很難靠自身找回來，因為稀疏獎勵無法在遠離目標的地方提供學習信號。

**問題二：狀態分佈崩塌**
即使偶爾到達目標，replay buffer 中大量狀態來自「曾待在目標附近」。策略學會在目標附近微調，但從初始狀態出發時不知道怎麼做——因為它從沒見過「從起點走到終點」的完整訓練信號。

如魚類控制任務實驗所示，episode 長度從 1000 增至 50000，學習曲線從能收斂到幾乎學不到任何東西。

## 16.4 前向-後向強化學習

**Forward-Backward RL** 的核心思路：訓練兩個策略，交替運行，不需要外部重置。

$$\boxed{\text{前向策略} \pi_F \text{（完成任務）} \leftrightarrow \text{後向策略} \pi_B \text{（復原環境）}}$$

**算法流程**：

1. 初始化：從 $\rho_0$ 採樣初始狀態（僅此一次）
2. 滾出前向策略 $\pi_F$，執行 $H$ 步，用前向獎勵 $R_F$ 更新 $\pi_F$
3. 不重置，從當前狀態繼續滾出後向策略 $\pi_B$，執行 $H$ 步，用後向獎勵 $R_B$ 更新 $\pi_B$
4. 不重置，回到步驟 2，重複

其中後向獎勵 $R_B$ 獎勵到達初始狀態分佈的狀態。偶爾（非常低頻率）進行外部重置作為保險。

**部署**：丟棄後向策略，只部署前向策略 $\pi_F$。

**局限**：每次都從初始狀態重試，效率並非最優——從「靠近目標的中間狀態」練習往往比從起點練習更有效。

## 16.5 Metal 算法：重置到專家狀態分佈

**Metal** 算法改進了後向策略的目標：不必完全回到初始狀態，而是回到專家示範所訪問過的任意狀態。

**動機**：已知道理論上最優策略的狀態軌跡大致在「初始狀態到目標」這條路上。如果後向策略能把機器人送回這條路上的某個中間點，前向策略就能從那個點繼續練習——效率比每次從頭開始更高。

**實現**：

1. 收集少量專家示範 $\mathcal{D}_{\text{expert}}$
2. 訓練一個**判別器**（discriminator），區分「策略訪問的狀態」vs「專家訪問的狀態」
3. 後向獎勵 $R_B(s) = -\log(1 - D(s))$，即讓判別器誤認為這是專家狀態
4. 後向策略學會將機器人送回專家狀態分佈中的任意點

這等價於最小化策略狀態分佈與專家狀態分佈的散度（類似 adversarial imitation learning 的思路）。

**EARL 基準結果**：

| 算法 | 特點 | 表現 |
|---|---|---|
| SAC（無後向策略）| 基線 | 很差，無法收斂 |
| Oracle（頻繁重置）| 不切實際上界 | 最佳 |
| Forward-Backward | 無示範 | 中等 |
| Metal | 有少量示範 | 最接近 Oracle |

EARL 基準的設定：每 200,000 步才允許一次外部重置，評估部署策略品質。

## 16.6 任務循環：多任務自主學習

前向-後向框架把問題分成「做任務」和「撤銷任務」，但有時「撤銷」本身比「做」還難（例如折毛巾比攤開更難），導致前向和後向策略學習速度嚴重不對稱。

**任務循環**（Task Cycle）提供了另一種思路：與其來回做一個任務，不如學習**一組相關任務的循環**，根據當前狀態自動選擇下一個要練習的任務。

**例子**：咖啡製作機器人有三個任務：

- **T1**：把杯子放進咖啡機
- **T2**：撿起地上的杯子（T1 失敗後需要）
- **T3**：按下咖啡機按鈕（T1 成功後繼續）

根據當前狀態決定練習哪個任務，不需要固定的前向-後向順序。

**算法框架**：

```
初始化多任務策略 π(a|s, z_i)
重複：
  1. 觀察當前狀態 s
  2. 選擇當前狀態下可行的任務 z_i（預定義任務圖或模型提議）
  3. 滾出 π(a|s, z_i) 執行 H 步
  4. 用任務 i 的獎勵更新策略（或用 off-policy 算法更新所有任務）
  5. 不重置，回到步驟 1
```

**語言模型提議任務**：最近研究用 VLM（視覺語言模型）觀察當前場景，自動提議可以練習的任務。機器人看著廚房場景，VLM 說「你可以拿起香蕉、打開微波爐、或推動茄子」，系統選擇其中一個來練習。結合圖像子目標（diffusion model 生成目標圖像），獎勵函數自動根據目標圖像計算。

## 16.7 Single Life RL：部署後的在線適應

即使已訓練好策略，現實世界總有訓練時未遇過的情境。**Single Life RL** 考慮的是：

> 策略有一次機會，遇到未知情境時，如何在沒有重置的情況下自主恢復？

**挑戰**：傳統 RL 更新（如 TD 更新）在遇到卡住的狀態時，繼續對「到達目標」的獎勵梯度下降往往無效——因為策略從未見過從這種被困狀態到目標的路徑。

**兩種解決思路**：

**思路一：引導回熟悉狀態**

不用「到達目標」的獎勵引導，改用「回到訓練時訪問過的狀態」的獎勵。機器人卡住後，RL 更新引導它回到熟悉的狀態，從那裡再嘗試完成任務。實驗顯示，四足機器人翻倒後能靠這個機制站起來並繼續前進。

**思路二：在高層空間探索**

在關節空間（低層）探索效率太低，不如在**預訓練技能空間**（高層）探索。若有「向前走」「翻身站起」等預訓練技能，適應時嘗試不同技能比嘗試不同關節角度更有效，可以解決需要策略性「換策略」的情況。語言模型可在此提供常識推理（「遇到障礙應該繞路」）。

## 16.8 實務考量

**安全約束**：自主學習過程中必須考慮安全——馬達過熱、物品掉落、碰撞。可以學習一個「恢復策略」，當偵測到高風險狀態時自動啟動，把機器人帶回安全區域。這需要危險狀態的示例資料，是雞與蛋問題：要學習危險，需要先經歷危險。

**獎勵學習**：真實機器人通常也沒有自動獎勵信號。自主 RL 常搭配獎勵學習算法（見第八章），用學到的獎勵模型取代人工設計的獎勵函數，讓整個系統完全不依賴人類監督。

**現狀**：這些算法仍主要在研究層面，商業機器人公司目前並不普遍採用。但作為邁向真正自主機器人學習的方向，具有重要的長期意義。

## 16.9 章節總結

| 挑戰 | 解決方案 | 核心思路 |
|---|---|---|
| 每次嘗試需要人工重置 | Forward-Backward RL | 學一個「撤銷策略」自動復原 |
| 只重試起點效率低 | Metal 算法 | 重置到示範狀態分佈 |
| 單一前向/後向不平衡 | 任務循環 | 根據狀態選最適合的任務 |
| 部署後遭遇未知情境 | Single Life RL | 回到熟悉狀態或高層技能探索 |

自主機器人學習的終極目標：初始化一次，讓機器人自行練習到精通，就像人類幼兒學走路一樣，無需外部重置，無需持續監督。
