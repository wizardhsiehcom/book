# 第二大腦：用 LLM 把筆記蒸餾成書

> 一頁筆記,記錄一個結論:**在「都用 LLM 寫」的前提下,該怎麼讓知識真正留下來。**
> 這頁本身就是一次示範——它是從一串對話蒸餾出來的。

---

## 一、起點:MkDocs 書 vs LLM 知識庫

兩者不互斥,本質不同:

| | **MkDocs 書** | **LLM 知識庫(RAG / 對話)** |
|---|---|---|
| 本質 | 人讀的靜態網站 | 機器檢索的語料 |
| 用法 | 自己翻、自己連結 | 問問題,它生成答案 |
| 內容狀態 | 你寫死的、確定的 | 被切片、機率性重組 |

- **MkDocs 強在**:可信可追溯(沒有幻覺)、結構即資產、零依賴永久保存、離線免費。
- **MkDocs 弱在**:查詢笨(只有關鍵字)、維護手工、不會幫你推理。
- **LLM 知識庫強在**:自然語言問答、跨文件綜合、自動摘要;**弱在**會幻覺、答案不穩、難追溯、綁廠商。

---

## 二、真正的定位:書是輸出,筆記是輸入

工具優劣是假議題,**「會不會持續用」才是真議題**。

> **筆記是輸入,書是輸出。LLM 知識庫存「還沒整理的你」,MkDocs 放「整理好的結論」。**

| | **MkDocs =【書】** | **LLM 知識庫 =【筆記】** |
|---|---|---|
| 流向 | 產出(output / 成品) | 原料(input / 素材) |
| 蒸餾度 | 已蒸餾、結構化 | 未蒸餾、可以亂、可以碎 |
| 讀者 | 給**人線性讀** | 給**機器隨機查** |
| 動作 | 翻、瀏覽 | 問、檢索 |

兩者是**前後關係**:亂筆記丟 KB → 沉澱出觀點 → 寫成 MkDocs 的書。
**書是筆記的「出口」**;沒有出口,KB 只會變成更深的黑洞。

!!! warning "棄坑體質的自我提醒"
    筆記系統棄坑(例如 Obsidian)的死因不是工具,是 **「只進不出 + 整理代替閱讀」**。
    換成任何工具,不解決這兩點,一樣會棄。
    對「不會主動查」的人,**被動可瀏覽性 > 主動可查詢性**——KB 要你先「想起」才查得到,書讓你「看得到」而回看。

---

## 三、LLM 越強,知識庫貶值了嗎?

部分對,但要分清楚**貶值的部分**和**增值的部分**。

LLM 變強,抹平的是**「世界知識」**,抹不平的是**「你的知識」**。

| | 會隨 LLM 變強而**貶值** | 反而**增值** |
|---|---|---|
| 內容 | 公開、可被重新生成 | 私有、無法被重新生成 |
| 例子 | 教科書摘要、概念解釋、某書講什麼 | 你的理解、取捨、跨書連出的觀點、你的決定 |
| 原因 | 模型本來就會,存它幹嘛 | 模型再強也猜不到你腦裡想什麼 |

**確實在貶值的兩樣**:

1. 重述公開知識的筆記(模型自己會講,而且更全)。
2. RAG 的水電工程(手動切片、embedding)——context window 變大後,直接丟乾淨 Markdown 給它讀就好。

**反而增值的**:瓶頸從「模型能力」移到「你餵的 context 品質」。模型越強,一份裝著**你獨有判斷**的語料價值越高。

> **問問題越便宜,「你問題裡帶了多少別人沒有的 context」就越是你唯一的護城河。**

知識庫的定位被逼著從「圖書館」升級成「你第二人格的記憶」。

## 四、最省力的蒸餾工作流

核心體悟:**蒸餾,LLM 已經幫你做了一半——就在你問問題的當下。**
你問的問題 + 它的好答案,本身就是「半成品的書頁」。

### 三步循環(不需要新工具)

```text
1. 隨手記 → 記哪都行,別搬家(別逼自己集中化原料)
2. 問 LLM → 出現「喔這個好」的答案 = 訊號,存下這段對話(連你問的問題一起存)
3. 一個主題覺得「夠了」→ 丟進 Claude Code → /mkdocs-update <book> → 它寫成一頁
```

### 兩個讓蒸餾趨近零成本的招

1. **每本書開一頁 `inbox.md`**:還沒想清楚的就無腦貼進去(原文、對話片段、半句想法),之後再升級成正式頁。允許書裡有亂的地方 = 你願意往書裡丟。

2. **固定一句蒸餾 prompt**,貼上原料就跑:

    > 「把以下我的筆記和對話,整理成一頁繁體中文 MkDocs Markdown。**保留我的結論和判斷,標出哪些是我說的、哪些是補充的事實**。純事實能精簡就精簡。」

### 一條要守的紀律

蒸餾**要批次、要有觸發點**(「一個主題讀完/想完了」),不要連續做。
連續整理 = 又變成「整理代替閱讀」。**書頁是某個主題的「結案動作」,不是日更。**

---

## 一句話總結

- **KB 是草稿匣,MkDocs 是書架。** 草稿匣讓你無腦丟,書架逼你回頭看。
- 既然「寫」的成本被 LLM 抹平了,就選**「逼你能看見」**的那個。
- 你要存的不再是「知識」(模型已會),而是**「你的判斷與私有 context」**(模型不會)。
