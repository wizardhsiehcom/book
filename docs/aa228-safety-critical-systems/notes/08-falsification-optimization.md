# Lecture 08: Falsification through Optimization - Reading Notes

## Overview
This lecture introduces the first validation algorithms to check if a system satisfies a property, specifically focusing on **falsification** (finding scenarios where the system fails).

## Key Concepts
1. **Purpose of Falsification**
   - Systematically finding failures in simulation before real-world deployment.
   - Used to inform future design decisions (e.g., enhance sensors, update policies, revise requirements, train operators, recognize limitations, or abandon projects).

2. **Direct Falsification (Direct Sampling)**
   - **Method**: Run $m$ random rollouts of the system and filter for failures using Signal Temporal Logic (STL) specifications.
   - **Limitation**: Performs poorly for rare failure events.
   - **Geometric Distribution**: The number of rollouts before finding a failure follows a geometric distribution. Expected rollouts = $1 / P_{fail}$. For safety-critical systems where $P_{fail} \approx 10^{-9}$, it takes a billion simulations on average to find one failure.

3. **Disturbances & Bookkeeping**
   - To efficiently search for failures, we take control of the sources of randomness by rewriting the system using **disturbances**.
   - **Splitting**: We separate the deterministic and stochastic components of the agent, environment, and sensor.
     - For example, the sensor observation model becomes a deterministic function $o = f(s, x_o)$, where $x_o$ is a disturbance sampled from a disturbance distribution (e.g., $x_o \sim \mathcal{N}(0, \Sigma)$).
   - All disturbances ($x_a, x_e, x_o$) are packaged into a single disturbance $x$.
   - **Trajectory Distribution**: Defined by an initial state distribution, a disturbance distribution for each time step, and a rollout depth. 
   - **Nominal Trajectory Distribution**: The expected real-world behavior distribution.

4. **Fuzzing**
   - Instead of sampling from the nominal distribution, sample from a **fuzzing distribution** (e.g., one with increased perception noise) to make failures more frequent.
   - This provides a baseline falsification technique.
   - **Caveat**: The likelihood of the discovered failures must still be measured under the nominal trajectory distribution. Fuzzing might find extremely unlikely failures.

5. **Optimization-based Falsification**
   - Formulate falsification as an optimization problem to systematically search the high-dimensional space of initial states and disturbance sequences.
   - **Decision Variables**: The initial state $s_0$ and the sequence of disturbances $\mathbf{x} = (x_1, \dots, x_T)$.
   - **Objective Function**: Minimize "closeness to failure", usually by minimizing the **robustness** metric from STL.
   - **Constraint**: The trajectory is deterministically generated by a rollout of the step function given the chosen initial state and disturbance sequence.
   - **Issue**: Minimizing robustness without considering probability can lead to extremely unlikely "impossible" failure trajectories. This will be addressed in future lectures by adding likelihood to the objective.
