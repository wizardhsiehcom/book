## 本講主問題

本講探討當賽局存在不完全資訊（incomplete information）時，議價過程為何會產生延遲與破局（disagreement）。介紹了兩種主要的不完全資訊機制：篩選（screening）與訊號傳遞（signaling），並將這些概念應用於健康保險的逆向選擇（adverse selection）與中古車市場的檸檬市場問題。最後利用完美貝氏均衡（PBE）來求解這些市場中的匯聚（pooling）與分離（separating）均衡。

## 核心概念

| 概念 | 說明 | 書稿處理方式 |
|---|---|---|
| Complete Information Bargaining | 完全資訊下的議價，賣方提出報價。無延遲、無破局，賣方榨取所有交易利得。 | 作為對照基準，簡要回顧。 |
| Screening | 缺乏資訊的一方（如賣方）先行動，透過報價來「篩選」具備私有資訊的另一方（買方），觀察其是否接受。 | 詳細說明獨佔定價與需求曲線的對應關係。 |
| Interdependent Values | 買方的私有資訊不僅影響其自身估值，也影響賣方的成本。經典例子為健康保險。 | 作為重點應用，推導 $\beta < 1/2$ 時的市場失靈條件。 |
| Adverse Selection | 逆向選擇。當賣方提高價格時，願意購買的買方群體組成發生改變，留下的是成本更高的惡劣客群（如最不健康的人）。 | 結合數學期望值推導與直覺解釋。 |
| Signaling (Market for Lemons) | 具備私有資訊的一方（如賣方）先行動，其報價會傳遞出關於商品品質的訊號。 | 設定 $H, L$ 兩型，求解 Pooling PBE，並探討 Separating 的條件。 |
| Pooling PBE | 高品質與低品質賣方皆開出相同價格 $P^*$。買方無法從價格區分品質，依先驗機率更新信念並決定是否購買。 | 詳細列出均衡條件（$C_H \le P^* \le V^*$）與脫軌信念設定。 |

## 重要細節

- 定義（解概念／賽局元素／經濟量）：買方估值 $V$，賣方成本 $C$。
- 定理敘述與證明思路：
  - 獨佔篩選：$V \sim U[0,1]$，利潤為 $(P-C)(1-P)$，最佳價格 $P^* = (1+C)/2$。
  - 逆向選擇模型：$C \sim U[0,1]$, $V = \alpha + \beta C$。若買方願意以 $P$ 購買，則 $C \ge (P-\alpha)/\beta$。賣方服務這些買方的平均成本為 $1/2 [1 + (P-\alpha)/\beta]$。利潤對 $P$ 的導數包含係數 $1 - 1/(2\beta)$。若 $\beta < 1/2$，提高價格反而使每單位利潤下降，導致市場機制崩潰（只能賣給所有人或不賣）。
- 賽局實例（矩陣、賽局樹、payoff 設定；標明「依口語重建」或 `待補`）：
  - 買賣雙方議價賽局樹（完全資訊）：賣方選擇 $P \ge 0$，買方選擇 Accept/Reject。買方報酬 $V-P$，賣方 $P-C$。
  - 檸檬市場模型：品質分為 $L, H$。機率為 $1-\pi, \pi$。買方估值 $V_L, V_H$，賣方成本 $C_L, C_H$。條件：$C_L < V_L$, $C_H < V_H$。
- 課堂 MobLab 遊戲：規則、結果、講者詮釋：無。
- 經濟應用場景：
  - 認罪協商（Plea bargaining）：太好的條件反而讓檢察官懷疑證據不足。
  - 航空公司吃到飽機票：吸引到的是極度頻繁飛行的超級虧本客群。
  - 健康保險市場：只有最常看病的人才願意買高價保險。
  - 中古車市場（Market for Lemons）：二手車賣方比買方更清楚車況。
- 講者例子或直覺說明：
  - 解釋 $\beta$：如果 $\beta$ 很大，代表估值隨成本變化劇烈，逆向選擇不嚴重；如果 $\beta$ 很小，代表一點點價格變動就會篩選出成本極高的客群。
- 問答重點：
  - 學生問：既然賣給所有人，利潤會是零嗎？講者答：取決於 $\alpha$。若 $\alpha$ 夠大可能有利潤，但在真實醫療市場通常 $\alpha$ 太小導致無利可圖而徹底崩潰。
  - 學生問：巨災保險是否也是逆向選擇？講者答：那是因為風險高度相關（correlated risks，如颶風一來全區受災），與私人資訊導致的逆向選擇機制不同。
- 容易忽略的提醒：
  - Pooling PBE 中，高型別賣方的參與約束（$P^* \ge C_H$）是最難滿足的（binding constraint）。

## 解概念與均衡分析

- 解概念名稱：Perfect Bayesian Equilibrium (PBE)
- 形式化定義：序列理性（Sequential Rationality）+ 貝氏一致性（Bayes Consistency）。
- 求解程序或演算法：
  - 針對 Pooling PBE：假設雙方都出 $P^*$，求出買方預期估值 $V^* = \pi V_H + (1-\pi)V_L$。設定買方策略：$P \le P^*$ 買，$P > P^*$ 不買。設定脫軌信念（off-path belief）：$P > P^*$ 認為是 $L$。檢查低型別與高型別賣方有無誘因偏離（高型別要求 $P^* \ge C_H$）。
- 適用範圍與限制（本講講者實際說明者）：分析不完全資訊動態賽局。分離均衡中，買方不能在兩種價格下都必然購買，否則低型別賣方會偏離去報高價，因此需要混合策略。

### 導讀

本章探討現實中普遍存在的「議價破局與延遲」現象。在先前的完全資訊模型中，只要存在交易利得，雙方必定會立即達成協議；然而，一旦引入「不完全資訊」，報價行為本身就具有傳遞或挖掘資訊的功能。我們將透過「篩選（Screening）」與「訊號傳遞（Signaling）」兩種機制，分析醫療保險市場的逆向選擇（Adverse Selection）以及中古車市場的檸檬問題（Market for Lemons），並利用完美貝氏均衡（PBE）來求解這些不完全資訊賽局。
