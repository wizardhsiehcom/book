## 本講主問題

本講探討動態賽局中納許均衡（Nash Equilibrium）的侷限性，特別是它無法排除「不可信的威脅（non-credible threats）」。為了解決這個問題，本講引入了「逆向歸納法（Backward Induction）」作為解概念，要求玩家在未達到的偶然事件（unreached contingencies）中也必須做出最佳反應。同時透過序列版本的最後通牒賽局、性別大戰與斯塔克伯格（Stackelberg）產量競爭來展示其應用。

## 核心概念

| 概念 | 說明 | 書稿處理方式 |
|---|---|---|
| 不可信的威脅 (Non-credible threats) | 在某些納許均衡中，玩家藉由宣告對自己不利但在未發生節點上能威嚇對手的策略來獲益。由於一旦到達該節點，玩家不會真的執行該威脅，因此稱為「不可信」。 | 在導讀與賽局實例中重點解釋，對比一般 NE 與 Backward Induction 的差異。 |
| 未達到的偶然事件 (Unreached contingencies) | 賽局樹上因為玩家的前期策略選擇而實際不會被走到的資訊集或節點。 | 說明 NE 允許在這些節點上有次佳（suboptimal）的策略，而 Backward Induction 則要求所有節點都必須最佳化。 |
| 逆向歸納法 (Backward Induction) | 求解完美資訊擴展形式賽局的演算法。從賽局樹的末端開始，反向推導每個節點的最佳選擇，最終得到一個純策略納許均衡。 | 獨立小節，給出形式化定義、步驟與性質。 |
| 完美資訊 (Perfect Information) | 賽局中的每個資訊集都只包含單一節點，意即玩家在行動時確切知道賽局先前的所有歷史。 | 定義 Backward Induction 的適用範圍，並舉撲克與猜拳為非完美資訊的反例。 |
| 斯塔克伯格競爭 (Stackelberg Competition) | 序列式的產量競爭。先動廠商（領導者）可以透過選擇高產量來「影響」後動廠商（追隨者）降低產量。 | 作為經濟應用實例，詳細推導並對比古諾（Cournot）競爭的結果。 |

## 重要細節

- 定義（解概念／賽局元素／經濟量）：完美資訊有限擴展形式賽局（Finite extensive form game with perfect information）。策略在動態賽局中必須是完整的權變計畫（Complete contingent plan），如 Stackelberg 中 P2 的策略是一個函數 $Q_2(Q_1)$。
- 定理敘述與證明思路：任何完美資訊的有限擴展形式賽局都存在至少一個純策略納許均衡（透過逆向歸納法構造）。因此像西洋棋這類零和、完美資訊賽局，其賽局價值必定是 +1, 0 或 -1，且存在純的最佳策略（不需混合策略）。
- 賽局實例（矩陣、賽局樹、payoff 設定；標明「依口語重建」或 `待補`）：
  1. **$3 的最後通牒賽局（ Ultimatum Game）**：P1 提議分給 P2 多少（0, 1, 2, 3），P2 接受或拒絕。接受則 P2 拿提議金額，P1 拿剩餘；拒絕則皆為 0。（依口語重建）
  2. **觀察行動的性別大戰（Boston game with observed moves）**：P1 先選 C 或 R，P2 觀察後選 C 或 R。存在三個 NE，其中兩個包含不可信威脅。（依口語重建）
  3. **Stackelberg 競爭**：需求函數 $P(Q) = \max(1 - Q, 0)$，邊際成本 $c$。
- 課堂 MobLab 遊戲：最後通牒賽局（Ultimatum Game）。結果顯示人們常拒絕低於 50% 的提議（因為公平性或懲罰心理），而提議者預期到這點，通常會提議大於 0 但小於 50% 的金額。
- 經濟應用場景：斯塔克伯格產量競爭中先動優勢的來源（影響力），以及廠商透過「淹沒市場（flood the market）」的威脅來維持低產量（在單次賽局中是不可信威脅，需重複賽局才能支撐）。
- 講者例子或直覺說明：解釋 $f$ 是函數，$f(x)$ 是數值，提醒學生在定義動態賽局策略時必須寫出函數 $Q_2(\cdot)$，而不只是具體數字。
- 問答重點：為何西洋棋有純策略均衡但猜拳沒有？因為猜拳是同時行動（非完美資訊）。非可信威脅在什麼情況下可信？可以透過重複賽局來建立。
- 容易忽略的提醒：逆向歸納法的「輸出」是一個完整的策略組合（Strategy Profile），而不是最後走到的「報酬（Payoffs）」。

## 解概念與均衡分析

- 解概念名稱：逆向歸納法（Backward Induction）
- 形式化定義：從有限完美資訊擴展形式賽局的終端節點前一個節點開始，為該節點選擇使玩家報酬極大化的行動，然後將該節點視為終端節點（報酬為所選行動的結果），依此類推反向回推至根節點。
- 求解程序或演算法：從樹的最末端開始，決定最末行動者的最佳選擇。將這些最佳選擇的結果回傳到上一層節點。重複此過程直到抵達根節點。
- 適用範圍與限制（本講講者實際說明者）：適用於有限的完美資訊擴展形式賽局（Finite extensive form games with perfect information）。不能有包含多個節點的資訊集。
- 在解概念精煉鏈上的位置與前後關聯：它是對納許均衡的一種「精煉（Refinement）」，排除了那些依賴於未達節點上次佳行動的納許均衡（即排除了不可信威脅）。
