# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.
This file is the canonical source of project instructions for Claude Code, Codex, and Antigravity. Other agent entry points should stay thin and point back here.

## What this repo is

A personal reading-notes knowledge base made of independent [MkDocs](https://www.mkdocs.org/) + [Material](https://squidfunk.github.io/mkdocs-material/) projects. Content lives in `docs/<name>/`, per-book configs live in `configs/<name>.yml`, and each book builds to `book/<name>/html/` (git-ignored). `index.html` at the repo root is a card-grid launcher that links to all built books.

## Commands

```bash
# Build all books (also runs sync-assets.sh first)
./build-books.sh

# Build a single book
uv run mkdocs build -f configs/<name>.yml

# Serve a single book locally (hot-reload)
./serve-book.sh <name>

# Sync shared assets into per-book dirs (required before manual build)
./sync-assets.sh
```

**Prerequisites**: Python + `uv` installed. Run `uv sync` once to install dependencies.

## Structure

| Path | Purpose |
|------|---------|
| `docs/<name>/` | Markdown source for each book |
| `configs/<name>.yml` | Per-book MkDocs config |
| `book/<name>/html/` | Build output (git-ignored) |
| `docs/assets/` | Shared CSS, JS, fonts (source of truth) |
| `docs/<name>/assets/` | Per-book synced assets (git-ignored, generated by `sync-assets.sh`) |
| `font.yml` | Font family / weight config; drives CSS generation in `sync-assets.sh` |

## Adding a new book

Use the `/mkdocs-create` command, or manually:

1. Create `docs/<new-name>/` with a `README.md` and content pages.
2. Create `configs/<new-name>.yml` by copying an existing config; update `site_name`, `docs_dir`, `site_dir`, and `nav`.
3. Run `./sync-assets.sh` to populate `docs/<new-name>/assets/`.
4. Register the book in `index.html` by copying an existing `.card` block before the `<!-- 新增書籍時在此複製一個 .card 區塊 -->` comment.

## Content conventions

- **Language**: all prose in Traditional Chinese (繁體中文) unless the user writes in English.
- **One concept per page**; cross-link with relative paths.
- Standard Markdown + Mermaid; inline HTML is allowed when needed.
- Prefer `flowchart`, `sequenceDiagram`, or `graph` diagram types.

## Mermaid rules (mermaid@10)

- Subgraph labels with spaces, Chinese characters, parentheses, or commas **must be quoted**: `subgraph "My Label（說明）"`.
- Line breaks inside node labels: use `<br/>`, never `\n`. Example: `A["line one<br/>line two"]`.
- Node labels with special chars (colons, parentheses, `+`, `×`) must be wrapped in double quotes inside brackets: `A["Conv+BN+ReLU"]`.
- Rule of thumb: if a label contains any character outside `[A-Za-z0-9_]`, add quotes.
