# 導讀：為什麼 AI 工程師還需要 Binary Hacks

這本 mdBook 不是《Binary Hacks／黑客秘笈100選》的逐章改寫，而是把原書最值得帶到現代 AI 時代的底層觀念，重新整理成一套可讀、可查、可延伸的知識地圖。

核心判斷很簡單：AI 的表面看起來像是模型、框架與提示詞工程，但一旦進入推理延遲、訓練穩定性、部署相依、記憶體洩漏、數值爆炸、外掛相容性等問題，最後總會回到二進位格式、執行期行為、浮點數、同步原語與 profiling。這正是 Binary Hacks 的價值所在。

## 這本書要回答什麼

1. 為什麼高階框架愈成熟，底層知識反而愈重要？
2. 哪些 2006 年的低階技巧，到了 2026 年依然是 AI infra 的基本功？
3. 面對模型載入、JIT 編譯、GPU profiling、數值穩定性時，應該用什麼心智模型？

## 這本 mdBook 的重組方式

- 不照原書 100 個 Hack 順序走，而是依現代 AI 工程的問題空間重組。
- 每頁只保留一個核心主題，方便後續擴充。
- 所有內容均為原創整理；原書只作為概念來源與索引，不直接重現原文。

## 這份整理的定位

你可以把它視為一張「AI 時代底層觀念的導航圖」：

- 原書教你如何看見抽象層之下的機器世界。
- 這本 mdBook 則補上今天真正會遇到的 AI 場景。
- 兩者的交會點，就是能把問題講清楚、拆清楚、量清楚的工程能力。

下一頁的[全書地圖](00-map.md)會先把整體架構攤開，再進入各個主題。
